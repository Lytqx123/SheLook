# SheLook 演示路线

> 文档基线：2026-07-23。演示写入仅适用于 development/test，不能作为生产初始化或真实性能数据来源。

SheLook 的演示重点不是“生成一张图片”，而是一次视觉运营活动如何形成经营决策闭环：**活动目标 → 内容生产 → 质量门禁 → 效果判断 → A/B 验证 → 复盘沉淀**。本指南只描述当前已落地的界面与接口，不预设任何真实经营结果。

> 运行基线（2026-07-22）：development Compose 已验证 API、Worker、对象存储和反向代理就绪；演示前仍须根据实际环境确认外部生成提供方、身份认证和是否允许写入合成数据。

## 准备环境

启动本地依赖与应用后，访问 `http://localhost:3000`：

```bash
docker compose up -d
```

空数据库可以直接从“视觉运营活动”创建一条活动，按真实流程逐步补充商品、方案和素材。若仅用于开发演示，可在**本地非生产环境**写入隔离的合成样例数据：

```bash
docker compose exec backend python -m scripts.seed_data
```

该脚本生成带 `demo` 标记的租户、商品、占位图片、审核、预测、实验和指标记录；它不会调用真实 AI 供应商，也不应在生产库运行。需要重建本地演示数据时，先阅读 `scripts.reset_demo_data` 的保护说明，再显式确认清理操作。

若工作区根目录的 `mock图片` 已放入本地素材，开发版 Compose 还可使用下列命令将全部 PNG 上传到开发 MinIO，并写入 **6 月 1 日至执行当天**的确定性模拟商品、素材、预测、每日指标、可追溯效果事实与成熟 CTR 反馈标签：

```bash
docker compose exec backend python -m scripts.seed_mock_catalog
```

该脚本仅允许 `development/test` 环境运行，并以 `MOCK-2026-*` SKU 作为幂等标识；重复执行不会重复写入。导入的数据明确标注为 `mock-catalog-2026-v1`，只用于本地页面、数据接入和 CTR 闭环验证，不代表真实店铺、平台或模型表现。

## 推荐演示主线

### 1. 创建视觉运营活动

打开 `/campaigns`，创建一项活动并填写活动名称、目标市场、经营目标及目标指标。活动可先没有关联商品；活动详情会集中显示后续素材、审核、预测、实验、行动建议和复盘洞察，而不是把用户留在互不关联的功能列表中。

活动接口使用稳定的 `GET/POST /api/v1/campaigns`；现有 `/api/campaigns` 保留为兼容别名。列表、详情和关联资源均按当前租户隔离。

### 2. 完成商品与视觉生产

从活动的“下一步”进入 `/publish?campaignId=<活动 ID>`。在工作台创建或关联商品，选择视觉方案，生成候选图片或视频，并将通过的素材发布到后续判断链路。带有 `campaignId` 的工作台会把商品、方案和图片 ID 回写到活动，活动阶段随之推进。

演示时应关注方案为什么适合目标市场、素材是否满足业务目标，而非将生成结果当作已经投放的事实。未配置外部生成提供方时，系统会明确返回可用性或降级状态；不要把占位或降级结果描述为真实投放素材。

### 3. 处理质量门禁

打开 `/review`，查看待人工审核队列。审核人员可基于合规、技术质量和视觉质量处理通过、驳回或补充问题说明；活动详情会汇总待审、通过和驳回数量。这里体现的是“自动判定 + 人工复核”的门禁机制，人工结论不会被前端自动覆盖。

### 4. 将预测转为决策候选

打开 `/prediction?campaignId=<活动 ID>`。页面把 CTR 潜力、爆款概率、退货风险等模型输出整理为候选决策卡，展示证据、置信度、风险和下一步建议。可将两项满足条件的候选素材创建为 A/B 实验；预测是辅助判断，不等同于真实经营结果。

### 5. 验证与复盘

实验详情页 `/experiments/<实验 ID>?campaignId=<活动 ID>` 展示对照版本、流量结果与显著性信息。实验完成后，进入 `/flywheel?campaignId=<活动 ID>` 记录本次活动的学习结论、证据和建议动作；这些洞察会回到活动时间线，供下一次方案选择和复盘使用。

### 6. 回到行动优先首页

`/dashboard` 按角色优先展示待审核、被阻塞任务、运行中的实验和经营异常等待决策事项；图表与模型指标下沉到洞察区域，用于解释“为什么要做”，而不是代替行动。

## 演示边界

- 平台只展示已接入或已导入的数据；没有授权、数据不足或需要人工确认时，界面和接口会返回相应状态，不伪造执行成功或经营指标。
- 演示数据用于验证流程、租户隔离和界面联动，不能用于评估模型效果、供应商表现或实际营收。
- 生产演示应使用受控租户、已审批的身份登录与脱敏业务数据，并保留审核和审计记录。
