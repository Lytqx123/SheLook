# SheLook 演示路线脚本

启动环境：`docker compose up -d && docker compose run backend python scripts/seed_data.py`

打开：`http://localhost:3000`

---

## 路线1：全流程演示（发品→推荐→生成→质检→上线）

**时长**：3分钟 | **入口**：`/publish`

```
1. [0:00] 发品工作台页面，展示空白上传区
   解说："SheLook的发品工作台，跨境商家只需上传一张平铺图"

2. [0:30] 拖拽连衣裙平铺图到上传区，点击"下一步"
   解说："支持JPG/PNG，最大5MB，上传即自动识别品类属性"

3. [0:45] 系统展示4套推荐方案（欧美街拍风、极简白底风、博主种草风、生活场景风）
   解说："AI分析了连衣裙的版型、材质、风格后，推荐了4套视觉方案"
   
4. [1:15] 勾选方案A+方案C，点击"开始生成"
   解说："选择欧美街拍风和博主种草风，系统将自动为不同市场生成变体"

5. [1:30] 生成进度展示：6张图排队生成，每完成一张显示缩略图+评分
   解说："采用双通道生成策略，Replicate主通道 + 本地SD降级保障"

6. [2:00] 全部生成完成，展示结果对比页
   解说："三阶段质量评估：L1合规检查 → L2技术评分 → L3美学评分，综合75分以上自动通过"

7. [2:30] 勾选3张通过的图片，点击"采纳发布"
   解说："审核通过的图片自动计算CLIP Embedding，可用于后续相似商品检索"

8. [2:45] 回到商品列表，显示"已发布"状态
   解说："至此，一次完整的发品流程就结束了，全程不到3分钟"
```

## 路线2：审核流程演示

**时长**：2分钟 | **入口**：`/review`

```
1. [0:00] 审核工作台页面，展示待审队列
   解说："当L2评分未达到自动通过阈值时，图片会进入人工审核队列"

2. [0:15] 队列按评分从低到高排序，最需要关注的排在前面
   解说："审核工作台支持批量审核，也可以逐张精细审核"

3. [0:30] 点击一张评分68分的图片，打开详情弹窗
   解说："这里可以看到五维度技术评分：清晰度72分、光影均匀65分、色彩丰富70分…"

4. [0:50] 指出问题维度（清晰度不足、材质还原度低）
   解说："光影均匀度65分，明显偏低，并且AI检测到材质还原度不够"

5. [1:10] 点击"驳回"按钮，输入驳回原因："光影不自然、材质偏差"
   解说："驳回后系统自动记录问题维度，触发模型重试，修正prompt参数"

6. [1:30] 对于通过的图片，点击"通过"即可
   解说："75分以上且各维度无短板的图片，一键通过，系统自动记录"

7. [1:45] 展示审核统计：今日通过率、驳回率、平均分
   解说："审核数据会汇总到看板，方便管理者追踪AI质检准确率"
```

## 路线4：A/B实验演示

**时长**：2分钟 | **入口**：`/experiments`

```
1. [0:00] A/B实验列表页面
   解说："A/B实验帮助量化验证不同视觉方案的实际效果差异"

2. [0:15] 展示历史实验列表：15个实验，标注状态和显著性
   解说："已完成15个A/B实验。绿色Tag'显著'表示p<0.05，有统计学意义"

3. [0:35] 点击某个实验查看详情：方案A CTR 4.8%，方案B CTR 3.2%，p=0.013
   解说："街拍风CTR 4.8% vs 极简风3.2%，p值0.013，差异有统计学显著性"

4. [1:00] 点击右上角"创建实验"，弹窗表单
   解说："创建新实验：选择商品、指定两版对比图片、设置流量比例"

5. [1:20] 填写实验参数：商品ID=1，版本A=图片3，版本B=图片12，流量50/50
   解说："设定50/50均分流量，系统在商品详情页随机展示两版图片"

6. [1:40] 创建成功，返回列表看到新的Running实验
   解说："实验启动后自动收集展示/点击/加购/转化数据，达到显著性后自动通知"
```

## 路线5：效果预测演示

**时长**：1.5分钟 | **入口**：`/review` → 图片详情

```
1. [0:00] 在审核工作台，点击一张已通过图片
   解说："每一张生成图片都会进行AI效果预测，帮助商家预判市场表现"

2. [0:15] 展示预测数据卡片
   解说："预测CTR 3.8%，80%置信区间 [2.7%, 4.9%]，这是基于GBDT模型+相似商品历史数据的预测"

3. [0:35] 爆款概率
   解说："Hit Probability 18%，代表该图片有18%概率成为爆款（CTR进入Top10%）"

4. [0:50] 退货风险评估
   解说："退货风险等级Low（绿色），结合CTR预测+爆款概率综合判定"

5. [1:10] 对比不同图片的预测结果
   解说："可以看到不同方案的预测数据差异明显，辅助商家做出更优选择"

6. [1:20] 展示特征重要性分析（如有时间）
   解说："模型告诉你哪些因素对CTR影响最大：色彩丰富度>构图平衡>光照均匀度"
```

## 路线6：双通道生图降级演示（技术亮点）

**时长**：1分钟 | **演示方式**：终端日志 + 截图

```
1. [0:00] 展示终端日志
   解说："这是SheLook在技术架构上的一个重要设计：双通道生图降级"

2. [0:10] 正常情况：Replicate API 调用日志
   解说："正常情况下，通过Replicate API调用FLUX.1模型，单张生成约8秒"

3. [0:20] 模拟降级：手动停止Replicate连接，展示switch逻辑
   解说："当Replicate API不可用时（超时/额度耗尽），系统自动降级到本地Stable Diffusion"

4. [0:35] 本地SD WebUI日志
   解说："切换到SD WebUI的txt2img API，虽然速度稍慢（约15秒/张），但不中断业务"

5. [0:45] 降级事件记录
   解说："每次降级事件都会记录到日志和metrics，方便运维监控"

6. [0:50] 技术优势总结
   解说："双通道+自动降级=99.9%生图服务可用性，消除对单一API供应商的依赖"
```
