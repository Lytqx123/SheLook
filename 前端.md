# SheLook 前端说明

## 1. 概览

一个给电商运营用的 Web 工作台，不是"数据看板只读展示"也不是"AI 工具单次输入输出"——运营要每天泡在上面的东西，所以优先考虑操作效率和信息密度，而不是花哨的动效。

目前走 B2B 办公风：深色侧栏、白色工作区、克制的配色、统一的卡片和低噪音图表。桌面端固定侧栏，移动端切抽屉导航。

## 4. 界面设计

### 4.1 色彩

以深蓝为主色，搭配低饱和辅助色：

- 主色：`#4F46E5`（只用关键按钮和选中态）
- 成功 `#0F9D75`、警告 `#D97706`、错误 `#DC2626`
- 背景 `#F5F7FB`、主文字 `#172033`、次文字 `#6B7280`

### 4.2 布局

- 桌面：264px 展开 / 80px 收起侧栏 + 自适应内容区
- 移动（767px 以下）：隐藏侧栏，304px 抽屉
- 内容区用预定义 CSS 类保持一致性：

| 类名 | 用途 |
| --- | --- |
| `office-workspace` | 纵向工作区 |
| `office-metric-grid` | 4 列指标卡片 |
| `office-kpi-strip` | 紧凑指标带 |
| `office-two-column` | 主次双栏 |
| `office-equal-columns` | 等宽双栏 |
| `office-form-grid` | 表单操作区 |

### 4.3 数据表格

统一风格：灰底表头、细分隔线、克制悬浮反馈、标签低饱和配色。数字列右对齐用等宽字体。

### 4.4 图表

信息优先：浅灰参考线、隐藏垂直网格、无粗轴线、白色 Tooltip。面积图用轻透明渐变，折线图默认隐藏节点仅悬停时显示。

> 先记一下：后续考虑要不要给聚类图加个导出功能

### 4.5 加载与空状态

- 路由切换用 Skeleton 骨架屏，不用空白页
- API 错误用 Alert / Message / Result + 重试入口
- 空列表用 Empty 占位
- 提交按钮异步期间 loading 防重复点击

### 4.6 移动端

767px 以下：
- 指标卡片 4→2→1 列
- 双栏变上下
- 图表图例自动收起
- 表格横向滚动

## 5. 数据请求

### API 客户端

所有 HTTP 请求通过 `src/lib/api.ts` 统一管理：自动带 Token、统一错误处理、正确处理 204 和空响应。不直接访问数据库或外部服务。

### 缓存和刷新

TanStack Query 管理服务端数据：

- 商品增删改发布后自动失效列表缓存
- 审核决策后自动失效队列缓存
- 实验变更后刷新列表和详情
- 生图任务进行中按间隔轮询状态
- 审核队列每 10s、实验列表和指标每 30s 自动刷新

### 实时推送

生图进度通过 WebSocket 推送（`/api/generation/ws/{image_id}`），生成完一张就能看到。断线时降级为轮询。

## 6. 状态管理

- 远程数据（商品、图片、审核、预测等）全用 TanStack Query
- 界面状态（侧栏展开/收起）用 Zustand 持久化到 localStorage
- 表单输入、筛选条件、弹窗开关留在页面组件内

## 7. 本地开发

```bash
cd frontend
npm ci
npm run dev
```

默认 <http://localhost:3000>。后端跑在宿主机时设环境变量：

```powershell
$env:BACKEND_URL='http://127.0.0.1:8000'
npm run dev
```

## 8. 构建检查

```bash
npm run lint
npx tsc --noEmit
npm run build
```

生产构建用 Next.js standalone 模式，Dockerfile 打包为最小镜像，监听 3000 端口。

## 9. 当前边界

- 前端不直接访问数据库、对象存储或外部 AI 服务
- Token 存 localStorage（当前单体架构够用）
- 第三方服务状态由后端检查后传给前端展示
