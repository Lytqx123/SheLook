# SheLook 前端说明

> 文档基线：2026-07-23。页面能力以当前构建产物为准；第三方凭据均不会回显到浏览器。

## 定位与体验原则

前端是面向跨境电商运营团队的日常工作台。当前体验以简约、信息密度适中的办公化界面为基调：深色导航承载工作分区，正文优先呈现行动、任务和结论；图表用于解释数据，不替代用户需要执行的下一步操作。

产品信息架构围绕“视觉运营活动”展开。用户不必先理解商品、素材、审核、预测、实验等后台模块的边界，而是可以从一次具体的市场活动开始，逐步推进策略、生产、审核、预测、实验和复盘。

> 2026-07-23 验证基线：锁定生产依赖的 `npm audit` 为零漏洞，`npm run lint`、TypeScript 检查和 Next.js 16.2.11 生产构建均已通过；容器前端健康检查正常。本文只描述当前 UI 和服务端契约，不把未配置的外部供应商能力呈现为前端可用功能。

## 页面与导航

应用路由位于 frontend/src/app，主要分为以下三组：

| 分区 | 路由 | 当前用途 |
| --- | --- | --- |
| 身份入口 | /login、/login/callback | 根据后端配置展示开发账号、飞书登录或企业 SSO，并完成回调。 |
| 日常运营 | /campaigns、/dashboard、/products、/publish、/review、/tasks、/video-generate | 运营活动、行动式首页、商品与素材生产、审核和任务处理。 |
| 经营决策 | /prediction、/experiments、/supplier | 预测比较、A/B 实验、供应商协同。 |
| 分析与治理 | /fairness、/clustering、/metrics、/flywheel、/audit-logs、/integrations、/integrations/providers、/settings | 公平性、聚类、指标与模型、数据飞轮、审计追溯、店小秘集成、外部 API 配置和运行时配置。 |

根路径会进入运营首页。运营活动位于日常运营的首位，活动详情会展示商品、候选素材、审核、预测、实验、经营指标、时间线、洞见和推荐的下一步动作。

`/integrations` 仅对管理员展示。店小秘页面可新增、修改、校验和删除当前租户的连接；API Key、Secret 和 Token 都是只写字段，保存后页面只显示“已加密保存”和短指纹。管理员可从页面发起一次同步并查看运行记录、范围、接收/写入计数和明确结果；在缺少贵司已授权接口契约时任务会显示为“等待供应商契约”，不会伪造外部网络请求或同步数据。

`/integrations/providers` 是统一的外部 API 配置页，视频生成页也提供直达入口。它按当前租户展示 Kling、Runway、Replicate、Gemini、Shopee 和 Amazon 的能力、启用状态与配置版本；页面只提交新密钥，保存后不会读取、缓存或回显明文。配置写入后会使视频供应商状态、图片生成、AI 审核和平台同步在下一次请求/任务中立即使用新版本；删除或停用后则明确显示为不可用。

`/settings` 仅对管理员展示。它管理经批准的租户级业务运行时参数，并展示当前是部署默认值还是租户覆盖值；保存后的新版本会在后续 API 请求和后台任务读取时生效。基础设施、身份和加密根密钥不会出现在该页面。

## 关键交互链路

### 运营活动

运营人员可以创建活动并指定名称、商品、市场和目标。活动状态与当前阶段独立维护，便于区分“是否正在进行”和“正处于策略、审核还是实验环节”。活动详情从后端聚合关联资源，并按权限提供推进、查看或复盘入口。

### 内容生产到验证

发品工作台支持商品创建、方案推荐、图片与视频生成。进入预测页后，用户可以比较候选素材的预测指标、依据、风险和建议动作，并选择两张候选图直接创建 A/B 实验。实验详情和数据飞轮可以携带活动上下文返回复盘，从而保持一条连续的经营路径。

### 行动式首页

运营首页根据登录用户的角色调整标题、待办和信息重点：运营人员优先看到增长机会、待审核与实验进展；审核人员优先处理风险队列；分析与管理角色可以继续进入指标、模型、审计和治理模块。图表和趋势信息保留在洞察层，避免首页成为只读看板。

## 角色与访问体验

前端识别 admin、operator、reviewer、analyst、supplier 和 viewer 六类角色，依据后端返回的角色和附加权限筛选导航、按钮和操作入口。客户端的判断仅用于信息架构和体验控制，不能构成安全边界；所有数据读取、写入与跨租户访问仍由后端授权和数据库策略决定。

移动端在窄屏下会将固定侧栏切换为抽屉导航；桌面端支持侧栏折叠。当前主界面使用统一的页面标题、表格、表单、状态标签、错误边界和加载状态，避免不同模块产生风格割裂。

## 技术实现

| 领域 | 当前实现 |
| --- | --- |
| 框架 | Next.js 16 App Router、React 19、TypeScript。 |
| 组件与样式 | Ant Design 6、Ant Design Next.js Registry、Tailwind CSS 4、项目级主题令牌。 |
| 数据请求 | TanStack Query 5 管理查询缓存、失效刷新和写入状态。 |
| 客户端状态 | Zustand 管理侧栏等轻量界面状态。 |
| 图表 | Recharts 用于趋势、比较和结构性图表。 |
| 服务访问 | src/lib/api.ts 统一封装 API 请求；src/hooks/index.ts 封装 Query/Mutation；src/types/index.ts 维护接口类型。 |
| 反向代理 | Next.js rewrite 将 /api 转发至 BACKEND_URL；容器内默认使用 backend:8000。 |

应用入口通过 Providers 集中配置 Ant Design 主题与 QueryClient。数据列表以紧凑表格、筛选和状态信息为主，不使用为装饰而装饰的卡片化列表。

## 本地运行

推荐先通过根目录启动脚本启动后端依赖。独立运行前端时：

~~~powershell
Set-Location frontend
npm ci
Set-Item Env:BACKEND_URL http://localhost:8000
npm run dev
~~~

访问 http://localhost:3000。若后端处于开发模式且未开启认证，可使用开发账号入口；启用企业认证后，登录页只显示后端已完整配置的登录方式。

构建与静态检查：

~~~powershell
Set-Location frontend
npm run lint
npm run build
~~~

## 当前边界

- 前端不直接保存第三方模型或平台密钥，也不直接调用这些服务；它只向受权限保护的后端提交一次性写入字段，所有外部集成、加密与运行时读取均由后端处理。
- 数据图表只渲染后端返回的统计与预测结果，不在客户端虚构经营数据。
- 菜单隐藏和按钮禁用不能替代权限校验。任何敏感操作都必须以服务端返回结果为准。
