# SheLook 前端说明

## 1. 概览

SheLook 前端是一个面向电商运营场景的 Web 工作台。定位上，它不是一个数据看板的"只读展示"，也不是一个 AI 工具的"单次输入输出"——而是一个需要运营人员在上面持续工作的日常工具。因此设计上更看重操作效率、信息密度和视觉一致性，而非炫技式的交互效果。

当前界面采用 B2B 办公风格：深色功能导航、白色工作区、克制的色彩体系、统一的信息卡片和低噪音的图表语言。桌面端以固定侧栏为主布局，移动端自适应切换为抽屉导航。

## 2. 技术栈

| 技术 | 用途 |
| --- | --- |
| Next.js 16 | App Router 路由、SSR 和 standalone 构建 |
| React 19 | 组件与客户端交互 |
| TypeScript | 类型约束 |
| Ant Design 6 | 表格、表单、卡片、弹窗和反馈组件 |
| Tailwind CSS 4 | 原子样式和局部布局 |

| Recharts | 数据可视化图表 |

## 4. 界面设计体系

### 4.1 色彩

以深蓝色为主色，搭配低饱和辅助色，在办公场景下长时间使用不刺眼：

- 主色：`#4F46E5`，仅用于关键操作按钮和选中态。
- 功能色：成功 `#0F9D75`、警告 `#D97706`、错误 `#DC2626`。
- 中性色：页面背景 `#F5F7FB`、主要文字 `#172033`、次要文字 `#6B7280`。

### 4.2 布局

- 桌面端：264px 展开 / 80px 收起的左侧导航 + 自适应内容区。
- 移动端（767px 以下）：隐藏固定侧栏，304px 抽屉导航。
- 内容区通过预定义的 CSS 布局类实现一致性：

| 类名 | 用途 |
| --- | --- |
| `office-workspace` | 页面纵向工作区 |
| `office-metric-grid` | 4 列指标卡片区 |
| `office-kpi-strip` | 紧凑运营指标带 |
| `office-two-column` | 主次双栏面板 |
| `office-equal-columns` | 等宽双栏 |
| `office-form-grid` | 表单操作区 |

### 4.3 数据表格

所有数据表格遵循统一的设计语言：轻量灰底表头、细分隔线、克制悬浮反馈、标签低饱和配色。数字列右对齐并使用等宽数字字体，方便运营人员快速扫读。

### 4.4 图表

图表遵循"信息优先"原则：浅灰水平参考线、隐藏垂直网格、无粗轴线、白色轻阴影 Tooltip。面积图使用轻透明渐变而非大面积色块，折线图默认隐藏节点仅悬停时显示，避免图表看起来"花"而干扰数据阅读。

### 4.5 加载与空状态

- 路由切换时展示 Skeleton 骨架屏，而非空白页或全屏 Spin。
- API 错误通过 Alert、Message 或 Result 组件展示，附带重试入口。
- 空列表始终使用 Empty 占位，不会直接显示空白页。
- 提交按钮在异步操作期间进入 loading 态，防止重复点击。

### 4.6 移动端适配

767px 断点以下：
- 指标卡片从 4 列折叠为 2 列再到单列。
- 双栏面板全部折叠为上下布局。
- 图表图例自动收起。
- 数据表格启用横向滚动，保持列完整。

## 5. 数据请求架构

### API 客户端

所有 HTTP 请求通过 `src/lib/api.ts` 统一管理：自动附加认证 Token、统一解析错误信息、正确处理 204 和空响应。前端不直接访问数据库或外部服务，所有业务操作通过同源 `/api` 代理到后端。

### 缓存与刷新策略

使用 TanStack Query 管理服务端数据的缓存、刷新和失效：

- 商品创建/编辑/删除/发布后，自动失效商品列表缓存。
- 审核决策后，自动失效审核队列缓存。
- 实验变更后，同时刷新实验列表和详情。
- 生成任务在进行中时按间隔轮询状态变化。
- 审核队列每 10 秒、实验列表和指标统计每 30 秒自动刷新。

### 实时推送

图片生成进度通过 WebSocket 实时推送（`/api/generation/ws/{image_id}`），生成完一张图就能在前端立刻看到。WebSocket 断开时自动降级为轮询，不阻断任务查看。

## 6. 状态管理

### 服务端状态

商品、图片、审核、预测、实验、指标等所有远程数据由 TanStack Query 管理，不在组件内部维护请求状态。

### 本地状态

界面级状态（如侧栏展开/收起）使用 Zustand 持久化到 localStorage。表单输入、筛选条件和弹窗开关保留在页面组件内部，不放入全局 Store。

## 7. 本地开发

```bash
cd frontend
npm ci
npm run dev
```

默认地址 <http://localhost:3000>。后端运行在宿主机时需设置环境变量：

```powershell
$env:BACKEND_URL='http://127.0.0.1:8000'
npm run dev
```

## 8. 构建与检查

```bash
npm run lint
npx tsc --noEmit
npm run build
```

生产构建使用 Next.js standalone 模式，Dockerfile 将 standalone 输出和静态资源打包为最小运行镜像，容器监听 3000 端口。

## 9. 当前边界

- 前端不直接访问数据库、对象存储或外部 AI 服务，所有操作通过后端 API。
- 认证令牌存储在 localStorage，适用于当前单体应用架构。
- 第三方服务的可用性由后端检查后传递给前端展示，前端不自行假设服务状态。
