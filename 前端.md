# SheLook 前端说明

## 1. 概览

SheLook 前端是基于 Next.js App Router 的电商视觉运营工作台。应用通过同源 `/api` 请求访问 FastAPI，使用 TanStack Query 管理服务端状态，并通过 WebSocket + 轮询展示图片生成进度。

当前界面采用简约电商办公风格：深色业务导航、白色工作区、克制的蓝色主色、统一的信息卡片、低噪音图表和简洁数据表格。桌面端使用固定侧栏，移动端切换为抽屉导航。

## 2. 技术栈

| 技术 | 用途 |
| --- | --- |
| Next.js 16 | App Router、页面路由、SSR/静态输出和 standalone 构建 |
| React 19 | 组件与客户端交互 |
| TypeScript 6 | 类型约束 |
| Ant Design 6 | 表格、表单、卡片、弹窗、反馈和布局组件 |
| Tailwind CSS 4 | 页面内原子样式和局部布局 |

| Zustand 5 | 持久化界面状态 |
| Recharts 3 | 面积图、柱状图、折线图、环形图和散点图 |
| dayjs | 日期格式化与日期范围 |

## 4. 页面路由

| `/video-generate` | 视频生成 | 视频提供方状态和生成请求 |
| `/audit-logs` | 审计日志 | 日志筛选、详情和请求链路 |

## 5. 应用布局

`src/components/AppLayout.tsx` 负责登录后页面的统一外壳。

### 桌面端

- 左侧固定导航，展开宽度 264 px，收起宽度 80 px。
- 菜单按商品工作台、质量与审核、增长实验、经营洞察和系统设置分组。
- 顶部栏显示当前页面、服务连接状态和用户入口。
- 内容区最大宽度由各业务页面控制，背景使用轻量办公画布。

### 移动端

- 767 px 以下隐藏固定侧栏。
- 顶部菜单按钮打开 304 px 抽屉导航。
- 指标卡片、双栏面板、图表图例和表单自动折叠。
- 数据表格通过横向滚动保留字段完整性。

### 用户状态

- 登录结果保存于 `localStorage.shelook_auth`。
- API 客户端自动将 `access_token` 写入 `Authorization: Bearer ...`。
- 401 响应会清理本地认证信息。
- 主布局发现无认证信息时跳转 `/login`。
- 退出登录会清除本地认证信息并返回登录页。

## 6. 数据请求架构

### API 客户端

`src/lib/api.ts` 是所有 HTTP 请求的统一入口：

- API 基础路径固定为同源 `/api`。
- 默认发送 JSON。
- 自动附加 Bearer Token。
- 统一解析后端错误中的 `detail`。
- 正确处理 `204 No Content` 和空响应体。
- 文件上传使用 `FormData`，平台图片导出使用 Blob。

Next.js 在 `next.config.ts` 中将 `/api/:path*` 代理到 `BACKEND_URL`：

- Docker 内默认：`http://backend:8000`。
- 本地前端开发：设置 `BACKEND_URL=http://127.0.0.1:8000`。

示例：

```powershell
$env:BACKEND_URL='http://127.0.0.1:8000'
npm run dev
```

### TanStack Query

`src/hooks/index.ts` 封装页面使用的 Query 和 Mutation。

全局策略：

- `staleTime`：30 秒。
- 请求失败重试 1 次。
- 窗口重新聚焦时不自动刷新。
- React Query Devtools 仅在开发环境加载。

典型刷新策略：

- 商品创建、编辑、删除或发布后，使商品列表缓存失效。
- 审核决策后，使审核队列缓存失效。
- 实验变更后，同时刷新实验列表和详情。
- 指标写入或同步后，刷新指标统计。
- 生成状态在 `pending/processing` 时按间隔轮询。
- 审核队列每 10 秒刷新，实验列表和指标统计每 30 秒刷新。

### WebSocket

`useGenerationWebSocket` 连接：

```text
/api/generation/ws/{image_id}
```

收到消息后立即刷新对应生成状态；连接异常时页面仍可依赖 Query 轮询，不会阻断任务查看。

## 8. 状态管理

### 服务端状态

商品、图片、审核、预测、实验、指标等远程数据全部由 TanStack Query 管理。

### 本地界面状态

Zustand 当前保存侧栏折叠状态：

```text
localStorage key: shelook-ui-storage
```

表单输入、筛选条件和弹窗开关保留在页面组件内部，避免将短生命周期状态放入全局 Store。

## 9. 视觉规范

### 主题

主题配置位于 `src/app/providers.tsx`：

- 主色：`#2563EB`。
- 成功：`#0F9D75`。
- 警告：`#D97706`。
- 错误：`#DC2626`。
- 页面背景：`#F5F7FB`。
- 主要文字：`#172033`。

Ant Design 的 Card、Menu、Table、Statistic、Tag、Button、Input、Select、Modal、Tabs、Form 和 Descriptions 均有统一 Token。

### 页面布局

全局 CSS 提供以下业务布局类：

| 类名 | 用途 |
| --- | --- |
| `.office-workspace` | 页面纵向工作区 |
| `.office-metric-grid` | 4 列核心指标区 |
| `.office-kpi-strip` | 紧凑型运营指标带 |
| `.office-two-column` | 主次双栏面板 |
| `.office-equal-columns` | 等宽双栏面板 |
| `.office-form-grid` | 输入—操作型表单 |
| `.office-result-block` | 同步、检查等操作结果 |

### 数据表格

所有 Ant Design Table 使用同一套简约规范：

- 轻量灰色表头。
- 不使用粗外框和多层圆角。
- 行间仅保留细分隔线。
- 悬浮反馈克制。
- 标签采用低饱和背景。
- 数字列优先右对齐并使用等宽数字。
- 列较多时显式配置 `scroll.x`。

### 图表

Recharts 图表统一使用 `.office-chart`：

- 水平参考线为浅灰实线，隐藏无意义的垂直网格。
- 坐标轴不显示粗轴线和刻度线。
- Tooltip 使用白色轻阴影浮层。
- 图例使用小圆点和低饱和文字。
- 面积图使用轻透明渐变。
- 柱状图限制最大柱宽，减少色彩数量。
- 环形图使用中心汇总数字和独立图例。
- 折线图默认隐藏常驻节点，仅显示活动节点。

### 登录页

登录页使用 `public/login-office-bg.png` 作为电商视觉运营场景背景，并叠加深色渐变与半透明登录卡片。移动端隐藏大段介绍内容，保留完整登录功能。

## 10. 错误、加载和空状态

- 路由级 `loading.tsx` 提供 Skeleton/Spin 反馈。
- `error.tsx` 和 `global-error.tsx` 处理 Next.js 错误边界。
- `ErrorBoundary` 处理业务组件运行错误。
- API 错误通过 Ant Design Alert、Message 或 Result 展示。
- 空列表使用 Empty，不用空白表格代替。
- 提交按钮在 Mutation 期间进入 loading，避免重复操作。

## 13. 新增页面约定

1. 在 `src/app/(main)/<route>/page.tsx` 创建路由入口。
2. 将需要交互的主体放在同目录 Client Component 中。
3. 使用 `PageHeader` 统一页面标题。
4. 在 `AppLayout.tsx` 的菜单和 `pageTitles` 中登记页面。
5. 在 `types/` 定义接口类型，在 `lib/api.ts` 增加请求方法。
6. 在 `hooks/` 封装 Query/Mutation，不在页面中重复请求逻辑。
7. 使用已有 `office-*` 布局类和全局表格/图表规范。
8. 为耗时加载增加 `loading.tsx`，为无数据场景增加 Empty。
9. 完成后运行 lint、TypeScript 和生产构建。

## 14. 当前边界

- 前端不直接访问数据库、MinIO SDK 或外部 AI 服务，所有业务操作通过后端 API 完成。
- 页面显示的第三方提供方状态来自后端配置检查；前端不会假设服务必然可用。
- `NEXT_PUBLIC_API_URL` 不是当前 API 客户端的路由来源；浏览器请求始终使用同源 `/api`，代理目标由服务端 `BACKEND_URL` 决定。
- 认证令牌存储于 localStorage，适用于当前单体 Web 应用；若迁移到更高安全等级的公网环境，应评估 HttpOnly Cookie/BFF 会话方案。
