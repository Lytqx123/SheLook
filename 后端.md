# SheLook 后端说明

## 1. 概览

基于 FastAPI 的异步服务，给前端工作台提供 REST API 和 WebSocket。核心职责是串起"理解图片→生图→评质量→预测效果→学习反馈"这条链路。

耗时操作（生图、质检、模型重训等）走 Celery Worker 异步，Celery Beat 管定时调度。PostgreSQL 存业务数据和向量，Redis 管缓存/限流/任务队列，MinIO 管图片对象存储。

## 3. 产品能力与 API 分组

后端 15 个业务 Router 按产品模块组织，所有业务接口要求认证。

### 3.1 商品工作台

**商品管理**：分页查询、创建、编辑、删除、发布。商品和方案、生成图片、审核状态形成关联，一个视图看完整状态。

**视觉方案**：以图搜图 + 方案推荐。以图搜图用视觉特征检索相似历史方案，方案推荐融合商品属性、视觉相似度和历史表现。

**图片生成**：异步提交，返回任务 ID 后通过 WebSocket 获取进度。生成完自动触发质检。

**效果预测**：单图或方案批量预测。综合商品属性、视觉特征和历史指标，输出 CTR/爆款概率/退货风险。模型不可用时降级标注。

### 3.2 质量与审核

**审核工作台**：L1 合规→L2 技术质量→L3 审美评估。人工支持通过/驳回并记录原因。

**公平性分析**：统计肤色分布、基线对比、方案偏差检查。基线是运营可配项。

**供应商分析**：多维评分（质量/审美/品牌一致性/预测效果），和品类平均对比，历史报告可查。

### 3.3 增长实验

**A/B 实验**：手动/自动创建，UCB 流量分配，按维度拆分指标，统计检验判断差异显著性。

**数据飞轮**：手动触发指标回流和模型重训。样本标注保留来源信息，版本可回滚。

### 3.4 经营洞察

**数据看板**：经营概览、CTR 趋势、市场对比、风格洞察，以及模型健康度指标。

**聚类分析**：K-Means 或 HDBSCAN 对商品图聚类，t-SNE 降维后可视化。每个簇汇总关键指标。

**指标管理**：JSON 批量幂等写入，Shopee/Amazon 同步，外部 Listing 映射。

### 3.5 系统能力

**平台导出**：按 Amazon/天猫/TikTok 等规范自动裁剪。

**视频生成**：查询 Kling/Runway 配置状态，优先可用通道。

**认证**：开发环境简化登录，生产环境 OIDC Authorization Code + PKCE。

**审计**：关键操作记日志，按操作类型筛选，按请求 ID 追踪链路。

## 4. AI 能力架构

### 4.1 视觉理解

CLIP 把图片映射为 512 维向量存 pgvector。同一套向量能力服务以图搜图、图文匹配、聚类分析和标签降级。

### 4.2 图片生成

多通道降级：优先 Replicate 或 Google，主通道失败换备用，全失败任务标记失败。开发环境支持占位图验证流程。

### 4.3 质量评估

生成完自动触发：
- 合规检查：内容安全、品牌要求、C2PA 元数据
- 质量评分：像素级指标，明确阈值
- 审美评估：多维度启发式评分

### 4.4 效果预测

核心原则：训练和推理的特征构造方式完全一致。特征包括类目/市场编码、价格归一化、质量评分、CLIP 向量、历史指标滚动统计。

冷启动用降级值，预测结果保留模型版本和时间戳可追溯。

### 4.5 数据飞轮

每天凌晨聚合经营数据，按阈值标注正负样本。训练特征构建复用预测服务的抽取逻辑，保证特征空间一致。模型版本化管理，支持回滚。

## 6. 异步任务

耗时操作走 Celery：

**同步任务**：生图、质检、商品向量计算。

**定时任务**：
| 时间 | 任务 |
| --- | --- |
| 每天 02:00 | 聚合经营指标 |
| 每周日 03:00 | 触发模型重训 |
| 每天 04:00 | 自动创建 A/B 实验 |
| 每天 06:00 | 更新实验流量分配 |

Celery Worker 并发默认 4，单任务硬超时 600 秒。

## 7. 安全

### 认证
- 生产用 OIDC Authorization Code + PKCE，校验 issuer/state/code verifier/签名后签发 JWT
- 开发环境简化登录
- 管理员角色通过 OIDC Claim 映射

### 接口安全
- Redis 滑动窗口限流（默认 600 次/60 秒）
- 指标写入独立 API Key
- 外部图片下载：白名单 + 大小限制 + SSRF 防护
- C2PA 内容来源签名，生产强制校验

### 生产启动检查

生产启动时强检：必须启用 OIDC、关闭 Mock、配指标 API Key、启用并强制 C2PA、提供签名证书、非默认凭据。任意一项不通过拒绝启动。

### 审计

关键操作写审计日志（操作类型/执行人/请求 ID/时间戳），按请求 ID 可追踪完整链路。

## 8. 可观测性

- structlog 结构化日志（请求 ID、操作、状态、耗时）
- Prometheus 指标：请求量、延迟、活跃请求、质量通过率、预测漂移、生成耗时、队列长度等
- Grafana 三套看板（系统健康/模型监控/业务质量）
- Flower 看 Celery 任务状态
- 可选 OpenTelemetry 分布式追踪

## 9. 部署

Docker Compose 部署 11 个核心服务，提供 dev/staging/prod 三套环境。后端容器启动时自动跑迁移再启 Uvicorn。开发环境建议 Docker 起基础设施，后端在宿主机 reload 跑。

## 10. 当前边界

- 第三方 AI 服务依赖外部凭据和配额
- 模型预测效果和线上数据量正相关，冷启动降级
- 公平性基线是运营参数，不是法律结论
- 生产 Swagger 默认关，`DEBUG=true` 才开
