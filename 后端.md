# SheLook 后端说明

## 1. 概览

SheLook 后端是一个基于 FastAPI 的异步服务，为前端工作台提供 REST API 和 WebSocket 连接。它的核心职责不是简单的 CRUD，而是串联起"理解图片 → 生成图片 → 评估质量 → 预测效果 → 学习反馈"这条产品链路。

耗时操作（图片生成、质量评估、模型重训等）由 Celery Worker 异步执行，Celery Beat 负责定时调度。PostgreSQL 保存业务数据和视觉向量，Redis 承担缓存、限流和任务队列，MinIO 管理图片的对象存储。

## 2. 技术栈

| 分类 | 技术 |
| --- | --- |
| 运行环境 | Python 3.13 |
| Web 框架 | FastAPI、Uvicorn、Pydantic 2 |
| 数据访问 | SQLAlchemy 2 Async、Alembic 迁移管理 |
| 数据库 | PostgreSQL 17 + pgvector 向量扩展 |
| 缓存与队列 | Redis、Celery 5、Flower 任务监控 |
| 对象存储 | MinIO（公开桶 + 私有桶） |
| AI / ML | PyTorch、Transformers、CLIP、scikit-learn、HDBSCAN |
| 外部服务 | httpx、Google GenAI、Replicate |
| 安全 | PyJWT、OIDC、C2PA 内容签名 |
| 可观测性 | structlog、Prometheus、Grafana、OpenTelemetry（可选） |

## 3. 产品能力与 API 分组

后端 15 个业务 Router 按产品模块组织，所有业务接口统一要求认证：

### 3.1 商品工作台

**商品管理**：支持商品的分页查询、创建、编辑、删除和发布。商品与视觉方案、生成图片、审核状态形成了数据关联，运营可以在一个视图中看到商品从建档到上架的完整状态。

**视觉方案**：提供以图搜图和方案推荐两个入口。以图搜图通过上传图片或输入 URL 检索视觉特征相似的历史方案；方案推荐则融合了商品属性、视觉相似度和历史表现数据。推荐结果会附带参考依据，帮助运营理解为什么推荐这个方案。

**图片生成**：异步生成任务管理，提交后返回任务 ID，后续通过轮询或 WebSocket 获取进度。生成完成后自动触发质量评估流水线，运营无需手动触发质检。

**效果预测**：支持按单张图片或按方案批量预测。预测模型综合商品属性、图片视觉特征和历史经营指标，输出预估 CTR、爆款概率和退货风险。模型不可用或样本不足时返回降级值并明确标注。

### 3.2 质量与审核

**审核工作台**：自动审核逻辑分为三级——L1 合规（内容安全、品牌规范）、L2 技术质量（清晰度、曝光、色彩等像素指标）、L3 审美评估（构图、主体、商业可用性等多维评分）。人工审核支持通过和驳回，并记录原因。

**公平性分析**：统计不同市场和类目的肤色分布，与运营配置的基线对比。支持方案级别的偏差检查。公平性基线是运营可配置项，非法律结论。

**供应商分析**：对供应商素材进行多维度评分（质量、审美、品牌一致性、预测效果），与品类平均水平对比，支持按供应商 ID 查询历史报告。

### 3.3 增长实验

**A/B 实验**：支持手动和自动创建实验。UCB 流量分配策略在优化收敛和探索之间做平衡。实验指标按日期等维度拆分，通过统计检验判断版本差异是否显著。

**数据飞轮**：模型运营的入口。支持手动触发指标回流和模型重训。每次样本标注保留来源信息，模型版本可回滚。

### 3.4 经营洞察

**数据看板**：提供经营概览、CTR 趋势、市场对比和风格标签洞察。也展示模型自身的健康度指标，让数据侧和业务侧的信息在同一视图中呈现。

**聚类分析**：使用 K-Means 或 HDBSCAN 对商品图的视觉特征聚类，t-SNE 降维后前端可视化。每个簇汇总样本量、平均 CTR、退货率等指标，帮助运营发现视觉模式与经营效果的关联。

**指标管理**：支持 JSON 批量幂等写入和 Shopee/Amazon 平台同步。外部 Listing 映射将平台的 SKU 与内部商品关联。

### 3.5 系统能力

**平台导出**：按 Amazon、天猫、TikTok Shop 等平台的图片规范自动裁剪和 AI 标识处理。

**视频生成**：查询 Kling 和 Runway 的配置状态，优先使用可用通道。所有通道不可用时明确返回不可用状态。

**认证**：支持开发环境简化登录和生产环境 OIDC Authorization Code + PKCE。管理员角色通过 OIDC Claim 映射。

**审计**：关键操作记录审计日志，支持按操作类型筛选、查看详情，以及通过请求 ID 追踪完整操作链路。

## 4. AI 能力架构

### 4.1 视觉理解

使用 CLIP 模型将图片映射为 512 维特征向量，存储在 PostgreSQL 的 pgvector 扩展中。同一套向量能力同时服务于以图搜图、图文匹配、聚类分析和标签降级等多个场景，避免维护多套特征。

### 4.2 图片生成

生成服务采用多通道降级策略：优先使用 Replicate 或 Google 的图片模型，主通道失败时尝试备用通道，所有通道失败时任务标记为失败。开发环境支持带标识的占位图用于流程验证。生成完成后自动写入 MinIO 并记录提供方和模型信息。

### 4.3 质量评估

评估流水线在图片生成完成后自动触发：
- 合规检查：内容安全、品牌要求、C2PA 元数据。
- 质量评分：清晰度、曝光、对比度、色彩等像素特征，使用明确阈值。
- 审美评估：通过多维度启发式评分表达构图、主体、背景和商业可用性。

### 4.4 效果预测

预测服务的核心设计原则是"特征对齐"：训练时构造特征的方式与推理时完全一致，避免线上线下特征不一致导致的预测偏差。特征包括：商品类目和市场编码、价格区间归一化、图片质量和审美评分、CLIP 视觉特征、历史经营指标滚动统计。

模型冷启动阶段使用配置中的降级值，预测结果中保留模型版本和时间戳用于可追溯。

### 4.5 数据飞轮

每天凌晨自动聚合各平台的经营数据，按阈值标注正负样本。训练特征构建直接复用预测服务的特征抽取逻辑，保证训练和推理的特征空间一致。模型工件版本化管理，支持查看历史版本和回滚。

## 6. 异步任务

耗时操作统一通过 Celery 异步执行：

**同步任务**：图片生成、质量评估、商品向量计算。

**定时任务**：
| 时间 | 任务 |
| --- | --- |
| 每天 02:00 | 聚合各平台经营指标 |
| 每周日 03:00 | 触发模型重训 |
| 每天 04:00 | 自动创建 A/B 实验 |
| 每天 06:00 | 更新实验流量分配 |

Celery Worker 并发数默认 4，单任务硬超时 600 秒，避免单个任务长时间占用 Worker。

## 7. 安全设计

### 认证
- 生产环境使用 OIDC Authorization Code + PKCE，后端校验 issuer、state、code verifier 和签名后签发 JWT。
- 开发环境支持简化登录流程，加速本地调试。
- 管理员角色通过 OIDC Claim 映射，权限模型当前区分管理员和普通用户。

### 接口安全
- Redis 滑动窗口限流（默认 600 次/60 秒）。
- 指标写入使用独立 API Key，与用户认证分离。
- 外部图片下载采用域名白名单、大小限制和私网地址阻断，防范 SSRF 攻击。
- C2PA 内容来源签名：生成内容携带来源信息，生产环境强制校验。

### 生产启动检查

生产环境启动时执行强制安全检查：要求启用 OIDC、关闭生成 Mock、配置指标 API Key、启用并强制 C2PA、提供签名证书，以及使用非默认安全凭据。任意一项不通过则拒绝启动。

### 审计

关键业务操作写入审计日志，每条日志包含操作类型、执行人、请求 ID 和时间戳。通过请求 ID 可追踪一个操作从发起到完成的完整链路。

## 8. 可观测性

- **结构化日志**：请求 ID、操作、状态和耗时使用结构化字段输出。
- **Prometheus 指标**：请求量、延迟、活跃请求数，以及质量通过率、预测漂移、生成任务耗时、队列长度等业务指标。
- **Grafana 看板**：系统健康、模型监控和业务质量三套看板。
- **Flower**：Celery 任务执行和 Worker 状态监控。
- **OpenTelemetry**：可选配置，启用后导出分布式追踪。

## 9. 部署

通过 Docker Compose 部署，包含 PostgreSQL、Redis、MinIO、后端 API、Celery Worker/Beat、Flower、前端、Nginx、Prometheus 和 Grafana 共 11 个核心服务。提供开发、预发和生产三套环境模板。

后端容器启动时自动执行数据库迁移，成功后再启动 Uvicorn。开发环境建议使用 Docker 启动基础设施（PostgreSQL、Redis、MinIO），后端在宿主机以 reload 模式运行以便调试。

## 10. 当前边界

- 第三方 AI 服务的能力取决于外部凭据和配额，未配置时对应功能降级。
- 模型预测效果依赖于已积累的真实数据，冷启动阶段使用降级策略。
- 公平性基线是运营参数，不构成法律结论。
- 生产环境的 Swagger 文档默认关闭，仅在 `DEBUG=true` 时开启。
