# SheLook 后端说明

## 1. 概览

SheLook 后端是一个基于 FastAPI 的异步业务服务，负责商品、视觉方案、图片生成、质量审核、预测、实验、指标、供应商分析、审计和认证等能力。

耗时任务由 Celery Worker 执行，Celery Beat 负责定时调度；PostgreSQL 保存业务数据和视觉向量，Redis 用于缓存、限流、任务队列和实时消息，MinIO 保存图片与受保护资源。

## 2. 技术栈

| 分类 | 技术 |
| --- | --- |
| Runtime | Python 3.13 |
| Web | FastAPI 0.139、Uvicorn、Pydantic 2 |
| ORM | SQLAlchemy 2 Async、asyncpg、psycopg2 |
| 迁移 | Alembic |
| 数据库 | PostgreSQL 17、pgvector |
| 缓存/队列 | Redis、Celery 5、Flower |
| 对象存储 | MinIO |
| AI/ML | PyTorch、Transformers、CLIP、scikit-learn、HDBSCAN、NumPy、Pillow |
| 外部客户端 | httpx、Google GenAI、Replicate |
| 安全 | PyJWT、cryptography、OIDC、C2PA |
| 可观测性 | structlog、Prometheus Client、OpenTelemetry |
| 包管理 | uv + `uv.lock` |

可选依赖组：

- `providers`：Google GenAI、Replicate、C2PA SDK。
- `observability`：OTLP 导出和 Redis/httpx/Celery Instrumentation。
- `dev`：Ruff、mypy、pytest、aiosqlite。
- `perf`：Locust。

## 4. 应用启动与生命周期

`app/main.py` 创建 FastAPI 应用并在生命周期中执行：

### 启动

1. 初始化结构化日志。
2. 在生产环境执行安全配置检查。
3. 生产环境预加载 CLIP 模型。
4. 初始化 Redis Pub/Sub。
5. 按配置初始化 OpenTelemetry。
6. 启动业务指标上报后台协程。

### 关闭

1. 停止指标上报协程。
2. 断开 Redis Pub/Sub。
3. 释放 SQLAlchemy 数据库连接池。

应用还注册了：

- CORS。
- 统一异常处理。
- 请求日志、请求 ID、审计和 Prometheus 中间件。
- Redis 滑动窗口限流。
- `/metrics` Prometheus ASGI 应用。

Swagger 和 ReDoc 仅在 `DEBUG=true` 时启用。

- `compliance_checker.py`：合规检查。

- `vision_reward.py` / `reward_scorer.py`：多维审美评分。
- `image_text_matcher.py`：图文匹配。
- `image_export_service.py`：平台规格转换与 AI 标识。
- `c2pa_service.py`：C2PA 签名和验证。

### 业务决策

- `auto_reviewer.py`：自动审核。
- `predictor.py`：CTR、爆款和退货风险预测。
- `experiment_auto_service.py`：自动实验和流量策略。
- `attribution.py`：指标归因。
- `data_flywheel.py`：样本回流、标注、重训和版本。
- `fairness_service.py`：肤色分类、分布和偏差检查。
- `supplier_report.py`：供应商报告。

### 集成与基础设施

- `storage_service.py`：MinIO 公共/私有对象和预签名 URL。
- `image_fetcher.py`：安全远程图片下载。
- `metrics_collector.py`：手工和平台指标采集。
- `metrics_reporter.py`：业务 Prometheus 指标。
- `pubsub.py`：Redis Pub/Sub。
- `video_generator.py`：Kling/Runway 视频通道。

## 9. 图片生成链路

```text
POST /api/generation
  → 创建 GeneratedImage 任务记录
  → Celery generation task
  → 根据类目选择提供方/模型
  → Replicate 或 Google
  → 失败时尝试 SD WebUI
  → 开发环境可生成明确标识的 Mock
  → 上传 MinIO
  → L1/L2/L3 质量处理
  → 更新数据库状态
  → Redis Pub/Sub 推送 WebSocket 事件
```

主模型路由当前包括：

- 默认：Replicate 上的 `black-forest-labs/flux-2-pro`。
- `promo`、`banner`：Google `gemini-3.1-flash-image`。

只有配置相应凭据后通道才会执行。生产环境禁止 Mock，通道全部失败时任务状态为失败。

## 10. 预测与模型管理

预测服务组合：

- 商品类目、市场和价格等业务属性。
- 图片质量和审美特征。
- CLIP 视觉特征。
- 历史经营指标。

结果包括预测 CTR、置信区间、爆款概率和退货风险。模型版本保存在运行时模型目录，API 可查看版本并执行回滚。

数据飞轮复用在线推理的特征抽取逻辑，避免训练与推理字段不一致。没有可用模型或输入不足时返回配置的降级值，不伪造模型置信度。

## 11. 指标采集

### 批量写入

`POST /api/metrics/batch` 接收指标数组并执行幂等 Upsert。单次写入数量和字段合法性由 Schema 与服务层校验。

### 平台同步

- Shopee：使用 Open API v2 凭据。
- Amazon：使用 SP-API Sales and Traffic 报告。
- Lazada：当前前端引导使用批量导入，后端未提供真实同步实现。

系统仅保存平台实际提供的字段，不推导缺失的点击、CTR 等数据。

### 外部映射

`ExternalListingMapping` 将平台、店铺、Listing/SKU 与内部商品或图片关联，为平台指标归因提供稳定键。

## 12. Celery

Worker 加载以下任务模块：

- `generation_task`
- `evaluation_task`
- `flywheel_task`
- `experiment_task`
- `vector_task`

运行参数：

- JSON 序列化。
- Asia/Shanghai 时区配置。
- 单任务硬超时 600 秒，软超时 540 秒。
- `worker_prefetch_multiplier=1`，避免长任务预取造成分配不均。
- 默认 Worker 并发数为 4。

Beat 计划：

| 调度 | 任务 |
| --- | --- |
| 每天 02:00 | `sync_daily_metrics` |
| 每周日 03:00 | `retrain_models` |
| 每天 04:00 | `auto_create_experiments_task` |
| 每天 06:00 | `update_traffic_allocation_task` |

## 13. 认证与权限

### 开发环境

`ENABLE_AUTH=false` 时，`/api/auth/token` 提供开发登录流程，便于本地调试。

### OIDC

`ENABLE_AUTH=true` 时使用：

1. 前端读取 `/api/auth/config`。
2. 调用 `/api/auth/login` 获取授权地址、state 和 PKCE 参数。
3. 身份提供方回调前端 `/login/callback`。
4. 前端调用 `/api/auth/callback`。
5. 后端校验 issuer、state、code verifier、audience 和签名后签发系统 JWT。

管理员角色由 `OIDC_ROLE_CLAIM` 和 `OIDC_ADMIN_ROLES` 映射。

## 14. 安全

### 生产启动检查

`APP_ENV=production` 时，后端拒绝以下配置：

- 使用开发默认 `SECRET_KEY` 或不安全存储凭据。
- 未启用 OIDC 或缺少 OIDC issuer/client 配置。
- `ALLOW_GENERATION_MOCKS=true`。
- 外部图片允许域名为空。
- 配置信任私网图片主机。
- 缺少 `METRICS_API_KEY`。
- 未启用/强制 C2PA。
- C2PA 证书或私钥路径为空、文件不存在。

Replicate 未配置会产生警告；Gemini 未配置会记录能力降级信息。

### 外部图片下载

`image_fetcher.py` 实施：

- 允许域名白名单。
- DNS/IP 检查和私网地址阻断。
- 最大响应体限制。
- 超时和重定向次数限制。
- 配置 MinIO URL 通过对象映射读取，而不是放宽通用 SSRF 规则。

### 限流

Redis 滑动窗口限流默认开启，默认容量 600 请求/60 秒。健康检查可绕过业务限流，确保编排系统能够探测服务状态。

### C2PA

C2PA 支持生成内容来源签名和审核时验证。生产环境要求证书、私钥、算法和强制校验均已配置。

## 15. 对象存储

| 配置 | 用途 |
| --- | --- |
| `MINIO_BUCKET` | 可公开展示的商品图片 |
| `MINIO_PRIVATE_BUCKET` | 受保护原始资产和签名内容 |
| `MINIO_PUBLIC_BASE_URL` | 浏览器可访问的对象外部地址 |
| `MINIO_PRESIGNED_URL_EXPIRY_SECONDS` | 私有对象预签名有效期 |

Nginx 将 `/images/*` 重写到公开桶，并设置 30 天不可变缓存。私有对象由后端生成预签名 URL。

## 16. 日志与监控

### Prometheus

基础指标：

- 请求总数。
- 请求延迟。
- 活跃请求数。

业务指标：

- 分层质量通过率。
- 预测漂移。
- 生成任务耗时。
- Celery 队列长度。

### 结构化日志

请求 ID、操作、状态、耗时和异常使用结构化字段输出，关键业务操作同时写入审计表。

### OpenTelemetry

设置 `OTEL_ENABLED=true` 和 `OTEL_EXPORTER_OTLP_ENDPOINT` 后启用分布式追踪。相关导出器位于 `observability` 可选依赖组。

## 20. 当前边界

- 外部提供方和电商平台接口由第三方账号、权限、配额和网络决定。
- 未配置真实图片提供方时，只有开发环境可以使用明确标识的 Mock。
- 视频提供方不可用时返回不可用/失败结果，不生成伪视频。
- Swagger/ReDoc 在生产环境默认关闭。
- 公平性基线需要业务和合规团队确认。
- 模型预测能力依赖真实指标和模型工件；冷启动阶段会使用配置降级值。
