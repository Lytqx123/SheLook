# SheLook 后端说明

> 文档基线：2026-07-23。当前 Alembic 迁移头为 `019`，租户隔离、运行时配置和外部业务 API 配置均由后端强制执行。

## 服务定位

后端为 SheLook 提供商品、素材、审核、预测、实验、指标、运营活动、组织权限、企业数据接入和审计能力。它采用模块化单体加异步任务的方式：同步 API 负责身份、权限、业务状态与查询；耗时的生成、向量处理、实验更新、店小秘同步、数据飞轮和事件分发交由 Celery 队列执行。

当前服务入口为 backend/app/main.py，数据库迁移由 Alembic 管理，当前迁移头为 `019`：`017` 增加版本化运行时配置，`018` 增加统一经营事实、真实效果事实、预测快照和 CTR 反馈标签，`019` 增加外部业务提供商的租户级加密配置。

## 技术与运行组件

| 领域 | 当前实现 |
| --- | --- |
| Web API | Python 3.13、FastAPI、Pydantic 2。 |
| 数据访问 | SQLAlchemy 2 Async、asyncpg、psycopg2、Alembic。 |
| 关系与向量数据 | PostgreSQL 17、pgvector。 |
| 对象存储 | MinIO，保存素材及相关对象。 |
| 异步任务 | Celery 5、Redis Broker/Result Backend、Celery Beat、Flower。 |
| AI/分析 | PyTorch、Transformers、CLIP、scikit-learn、HDBSCAN；外部生图、视频、审核和平台适配器由租户 Web 配置启用。 |
| 可观测性 | Prometheus 指标、结构化日志、可选 OpenTelemetry。 |

Docker Compose 将任务拆分为 orchestration、generation、model 和 analytics 队列，避免素材生成等慢任务阻塞日常查询和审核操作。

2026-07-23 的容器复核还确认了 Celery prefork 与异步数据库连接的边界：`app.tasks.async_utils` 会在 Worker fork 后丢弃继承的连接池，并在每个同步任务的异步运行结束前释放连接。这样既避免父进程连接被子进程继承，也避免 `asyncio.run` 创建的新循环复用上一轮的 asyncpg 连接。Beat 已实际持续派发 Outbox，Worker 已持续成功消费；82 项后端回归测试、迁移 `019` 与完整性/RLS 门禁均通过。

| 生成与审核 | /api/generation、/api/video、/api/review | 图片/视频生成、状态查询、自动审核和人工审核队列。 |
| 决策与实验 | /api/prediction、/api/experiments | 效果预测、模型版本、A/B 实验、流量与结果分析。 |
| 运营活动 | /api/v1/campaigns | 视觉运营活动、阶段和状态推进、活动详情、洞见读写；兼容前缀 /api/campaigns 仍可使用。 |
| 指标与学习 | /api/metrics、/api/dashboard、/api/flywheel | 指标写入/同步、看板、样本沉淀、训练与模型版本。 |

| 外部 API 配置 | /api/provider-configs | Kling、Runway、Replicate、Gemini、Shopee、Amazon 的只写凭据、非敏感连接参数、启用状态、版本与本地加密校验；仅管理员或 `tenant:manage` 可维护。 |
| 运行时配置 | /api/runtime-settings | 白名单业务参数的版本化读取、更新、恢复默认和历史审计；仅管理员或具备 `tenant:manage` 权限的成员可维护。 |

## 视觉运营活动模型

视觉运营活动是产品层的聚合对象，由 VisualOperationCampaign 和 CampaignInsight 持久化。活动记录商品、市场、目标、阶段、状态、方案/素材/实验引用、负责人、推荐动作和下一步；活动详情在读取时聚合关联资源，返回素材数量、审核状态、预测摘要、实验结果、经营指标、行动项、时间线和洞见。

活动引用的商品、方案、素材和实验均在写入时校验租户归属和商品一致性。活动状态转移受服务端限制，避免归档活动被重新推进或跳过必要的业务状态。当前 API 支持列表、创建、详情、更新、状态更新及洞见读写。路由入口保留在 `app/api/campaigns.py`，共享鉴权/校验与详情聚合分别位于 `campaign_common.py`、`campaign_details.py`；该拆分不改变 `/api/v1/campaigns` 或兼容前缀 `/api/campaigns` 的访问契约。

## 多租户、认证与权限

每个业务数据域都携带 tenant_id。服务端从经过验证的访问令牌或开发环境上下文建立租户范围，成员关系和角色决定基本权限，额外 permissions 与 unit_ids 用于细粒度授权。当前预置角色为：admin、operator、reviewer、analyst、supplier、viewer。

企业身份支持通用 OIDC/SSO 和飞书 OAuth。外部身份只能证明用户是谁，不能直接授予本地租户权限；系统要求将外部组织标识显式映射到本地 tenant_id，并要求用户是有效成员。开发账号登录仅在非生产且未启用认证时开放。

店小秘连接配置不再依赖业务 API 环境变量。连接的 API 地址、店铺范围、同步范围和授权凭据在当前租户内保存；凭据使用后端加密后写入数据库，响应、审计和前端均不包含明文。运行时仅保留 `INTEGRATION_CREDENTIALS_ENCRYPTION_KEY` 这一启动级加密根密钥，生产环境缺失时拒绝保存连接。同步由独立 Celery 任务创建 `IntegrationSyncRun`；没有贵司已授权的接口地址、签名、字段契约、分页/游标及测试授权时，Provider Adapter 会把运行标记为 `awaiting_provider_contract`，不会发出猜测请求或报告同步成功。

同一安全边界已覆盖外部业务 API：`ProviderConfig` 以 `(tenant_id, provider)` 唯一保存加密凭据和非敏感参数，`provider_configs` 启用强制 RLS。Kling/Runway 视频生成、Replicate/Gemini 图片生成与 Gemini 审核、Shopee/Amazon 指标拉取均只在运行时从该表读取当前租户的已启用配置；旧的业务 API 环境变量和 Compose 注入已删除。数据库、对象存储、Redis、认证回调、监控入口和加密根密钥属于部署基础设施，仍必须在运行环境的 Secret 中保存，绝不能交由浏览器配置。

真实 CTR 数据遵循独立证据链：`ExternalEntityMapping` 将外部商品/刊登/创意显式关联内部商品和素材，`CommerceFact` 保存可幂等更新的经营事实，`PerformanceFact` 保存来源、曝光、点击和成熟时间，`PredictionSnapshot` 固化预测当时的模型/实体上下文，`ModelFeedbackLabel` 只由已映射、已成熟且达到阈值的效果事实生成。旧的 `DailyMetric` 仍为兼容读写保留，但不再由飞轮把“实际值”写进 `PredictionRecord`。

PostgreSQL 环境启用行级安全策略作为第二道隔离。生产启动会验证租户强制开关、RLS 开关以及数据库运行账号不具备超级用户或 BYPASSRLS 权限。配额、功能开关、成员与组织单元共同构成组织级运行边界。

## 可靠性与安全措施

- 工作流任务和 Outbox 事件用于记录异步动作、失败状态和后续分发；Celery Beat 负责周期性任务。
- Redis Pub/Sub 用于 WebSocket 场景；初始化失败时服务端记录降级状态，客户端可继续通过轮询读取。
- API 包含限流、请求链路标识、结构化日志与审计记录；外部图片抓取受允许主机、大小和私网访问限制控制。
- 生成链路支持 C2PA 内容凭证。生产环境要求启用并强制签名，同时配置证书与私钥。
- 生产启动校验认证、HTTPS 回调、显式租户映射、RLS、受限数据库账号、指标密钥、独立的集成凭据根密钥和关闭生成模拟等安全条件，缺失关键配置会拒绝启动。

## 数据与任务流

1. 同步 API 校验身份、权限、租户范围和业务参数。
2. 对需要长时间处理的操作创建任务或记录，并路由到相应 Celery 队列。
3. Worker 在显式租户上下文中执行生成、审核、向量索引、实验、指标或飞轮任务。
4. 结果写入 PostgreSQL、MinIO 和审计/工作流记录，前端通过查询接口获取状态和结果。
5. 指标和实验结论可沉淀为活动洞见，供后续运营活动和模型学习使用。

## 本地开发与验证

推荐通过根目录部署脚本准备依赖服务。单独开发后端：

~~~powershell
Set-Location backend

uv run alembic upgrade head
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
~~~

Set-Location backend
uv run ruff check .

uv run pytest
~~~

## 当前边界

- 第三方图像、视频与平台同步能力受凭据、配额、供应商状态和网络环境影响。开发环境可按显式配置启用生成模拟；生产环境禁止该模式。
- Compose 以开发配置为基线：源码挂载、诊断端口均只用于本机。预发、生产覆盖移除源码挂载与直接管理端口，只消费 digest 固定的镜像，并由 Nginx 作为唯一公开入口。
- API 和 Worker 启动不再执行 Alembic；`migrate` 服务才接收 `DATABASE_MIGRATION_URL`。运行服务只接收受限的 `DATABASE_URL`，避免 schema-owner 凭据扩散到常驻容器。
- 演示数据默认不填充，且 `seed_data.py`、`reset_demo_data.py` 只允许在 development/test 环境运行。开发环境需显式传入 `-SeedDemo`/`--seed-demo` 并完成二次确认。
- 预发、生产 Compose 会强制关闭生成模拟，挂载 C2PA 证书/私钥与 Prometheus 指标密钥为 Docker secret 文件；启动脚本会校验镜像 digest、非开发凭据、认证、CORS、C2PA、迁移连接和 secret 文件是否齐全，但不会输出密钥值。
- 预测值和策略建议是基于现有样本与规则/模型的辅助信息，不构成实际经营结果承诺。
- Docker Compose 已提供开发、预发布和生产覆盖配置；启动默认不写入演示数据，且预发、生产及直接脚本调用均被环境保护拒绝。生产发布前仍须完成目标环境的权限、密钥、存储、备份、观测与容量验证。
