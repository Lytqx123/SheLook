# SheLook 项目介绍

## 1. 项目定位

SheLook 是一套面向电商商品视觉运营的 AI 决策系统。它不只负责“生成一张商品图”，而是把商品资料、视觉方案、生成结果、质量审核、经营效果和模型反馈连接起来，为运营人员提供可追踪、可审核、可迭代的视觉生产工作台。

系统当前覆盖以下完整链路：

```text
商品建档
  → 相似图检索与视觉方案推荐
  → AI 图片生成
  → 合规、质量与审美评估
  → 人工审核与平台导出
  → CTR / 爆款概率 / 退货风险预测
  → A/B 实验与经营指标回流
  → 样本标注、模型重训与版本管理
```

SheLook 适合商品运营、视觉审核、供应商质量管理、增长实验和模型运营等岗位在同一套系统内协作。

## 2. 使用角色

| 角色 | 主要工作 |
| --- | --- |
| 商品运营 | 建立商品、选择视觉方案、发起生成、发布商品 |
| 视觉审核 | 查看自动质检结果、人工通过或驳回图片 |
| 供应商质量人员 | 分析供应商素材、查看维度得分和历史报告 |
| 增长运营 | 创建 A/B 实验、观察 CTR 差异、调整流量 |
| 数据/模型运营 | 导入经营指标、触发数据飞轮、查看模型版本和回滚 |
| 系统管理员 | 配置认证、外部服务、安全策略和监控 |

系统认证支持开发环境本地令牌流程，以及生产环境 OIDC Authorization Code + PKCE。当前权限模型区分管理员和协作成员；后端业务路由统一经过认证依赖。

## 3. 当前产品模块

### 3.1 商品工作台

#### 发品工作台

- 创建商品基础资料、SKU、类目、价格区间和目标市场。
- 通过图片 URL 或本地上传执行以图搜图。
- 基于 CLIP 向量和 pgvector 检索相似商品与既有方案。
- 使用商品属性、视觉相似度和历史表现进行方案推荐。
- 按方案发起异步图片生成，展示任务进度和生成结果。
- 对生成图片执行图文匹配、审美评估和质量检查。

#### 预测决策

- 按单张图片发起预测。
- 对同一商品的多个方案进行批量预测比较。
- 输出预估 CTR、爆款概率、退货风险和相关置信信息。
- 查看预测历史、模型版本并支持模型回滚。

#### 商品管理

- 商品分页查询和条件筛选。
- 创建、编辑、删除和发布商品。
- 查看商品、方案、生成图片和审核状态的关联信息。

### 3.2 质量与审核

#### 审核工作台

- 展示待审核图片队列。
- 触发自动审核。
- 执行人工通过或驳回，并记录原因。
- 查看质量分数、合规问题、审美维度和审核记录。

自动质量链路分为三个层级：

1. L1 合规检查：内容安全、品牌要求、元数据和 C2PA 状态。
2. L2 图像质量：清晰度、曝光、对比度、色彩、白边和其他像素级特征。
3. L3 审美评估：通过多维启发式评分表达构图、主体、背景、商业可用性等视觉质量。

#### 公平性分析

- 按市场和类目统计肤色分布。
- 将实际分布与运营配置的市场基线进行比较。
- 对指定方案执行偏差检查。
- 对未知分类进行单独处理，并限制单次新分类数量。

公平性基线是运营配置，不是法律或人口统计事实。正式上线前需要由本地数据、合规和法务负责人确认。

#### 供应商分析

- 分析供应商提供的图片 URL。
- 汇总质量、品牌、审美、预测和退货风险相关维度。
- 与品类标杆进行比较并给出改进建议。
- 按供应商 ID 查询历史分析报告。

### 3.3 增长实验

#### A/B 实验中心

- 创建、查看和停止实验。
- 查询实验详情和按日期等维度拆分的真实指标。
- 基于统计检验判断版本差异。
- 支持自动创建实验和 UCB 流量调整。

#### 数据飞轮

- 聚合线上经营指标并生成训练样本。
- 按阈值自动标注正、负和中性样本。
- 触发模型重训。
- 查看当前模型和历史版本。
- 将线上模型回滚到指定版本。

### 3.4 经营洞察

#### 数据看板

- 汇总生成量、审核通过率、CTR、CVR、营收和退货率。
- 展示近 30 天 CTR 趋势。
- 对比不同市场的 CTR、CVR 和素材数量。
- 查看风格标签的使用分布。
- 展示模型 AUC、高 CTR 预测占比和人工复核比例。

#### 聚类分析

- 使用 K-Means 或 HDBSCAN 对视觉向量聚类。
- 通过 t-SNE 输出二维坐标用于前端展示。
- 汇总各簇样本量、平均 CTR、退货率、主要类目和标签。

#### 指标数据管理

- 通过 JSON 数组批量写入每日经营指标。
- 同步 Shopee 和 Amazon 平台指标。
- 管理外部 Listing 与内部商品/图片映射。
- 查看指标总量、覆盖图片数和数据日期范围。

指标写接口使用 `X-API-Key`；开发环境可关闭该校验，生产环境强制配置。

### 3.5 系统能力

#### 平台导出

系统根据目标平台规格处理图片尺寸、背景、格式和 AI 标识。当前支持：

- Amazon
- 天猫
- TikTok Shop
- TikTok 方图
- Shopify

#### 视频生成

- 查询 Kling 和 Runway 提供方的配置状态。
- 优先使用可用的 Kling 通道，再尝试 Runway。
- 所有视频提供方不可用时明确返回不可用状态，不伪造视频结果。

#### 审计与链路

- 查询操作日志、日志详情和同一请求的完整链路。
- 使用请求 ID 关联商品、图片、生成、审核和预测操作。
- 对关键业务请求输出结构化日志和 Prometheus 指标。

## 4. AI 与数据能力

### 4.1 向量检索

- 使用 `openai/clip-vit-base-patch32` 生成 512 维视觉向量。
- PostgreSQL 通过 pgvector 保存向量。
- HNSW 索引用于相似图片检索。
- 同一向量能力同时服务于以图搜图、标签降级、图文匹配和聚类。

### 4.2 方案推荐

系统提供两类推荐入口：

- 视觉推荐：根据输入图片检索相似方案。
- 融合推荐：组合商品属性、视觉特征和历史表现进行排序。

推荐结果包含方案信息、相似度/推荐分和用于解释的关联数据。

### 4.3 图片生成

图片生成由统一服务编排：

1. 根据类目选择主通道和模型。
2. 尝试已配置的 Replicate 或 Google 图片服务。
3. 主通道失败时尝试 SD WebUI。
4. 开发环境允许返回明确标识的占位图，用于验证异步流程。
5. 生产环境禁止生成 Mock；所有真实通道失败时任务失败。

生成结果写入 MinIO，并在数据库保存提供方、模型、任务状态、质量分数和审核状态。

### 4.4 标签与视觉评估

标签提取会根据可用配置依次使用外部视觉模型和本地 CLIP 能力。图文匹配通过图片向量与文本向量的相似度给出匹配结果；审美评估输出多维分数和综合结论。

### 4.5 效果预测

预测服务将商品属性、图片质量、视觉特征和历史指标组合为模型输入，输出：

- 预测 CTR。
- CTR 置信区间。
- 爆款概率。
- 退货风险等级。
- 模型版本与预测时间。

模型不可用或样本不足时，服务使用配置中的安全降级值，并在结果中保留可追踪信息。

### 4.6 指标回流

每日指标以图片、日期和来源平台为主键维度进行幂等写入。平台同步不补造平台未提供的指标；例如 Amazon 报告没有点击或 CTR 时，系统不会推导伪造值。

## 5. 系统架构

```text
┌─────────────────────────────────────────────────────────┐
│                    Next.js 前端                         │
│ 发品 / 商品 / 审核 / 预测 / 实验 / 看板 / 系统管理      │
└──────────────────────────┬──────────────────────────────┘
                           │ REST + WebSocket
┌──────────────────────────▼──────────────────────────────┐
│                    FastAPI API                          │
│ 认证、限流、审计、业务路由、服务编排、Prometheus 指标    │
└──────────────┬──────────────┬──────────────┬────────────┘
               │              │              │
       ┌───────▼──────┐ ┌────▼─────┐ ┌──────▼───────┐
       │ PostgreSQL   │ │  Redis   │ │    MinIO     │
       │ + pgvector   │ │ Cache /  │ │ Public /     │
       │              │ │ Queue /  │ │ Private      │
       └──────────────┘ │ PubSub   │ └──────────────┘
                        └────┬─────┘
                             │
                  ┌──────────▼──────────┐
                  │ Celery Worker/Beat  │
                  │ 生成、评估、向量、  │
                  │ 实验与数据飞轮任务  │
                  └─────────────────────┘
```

### 5.1 前端

- Next.js App Router 和 React Client Components。
- Ant Design 提供表单、表格、弹窗和反馈组件。
- TanStack Query 管理服务端数据、缓存、轮询和失效刷新。
- Zustand 保存侧栏等轻量界面状态。
- Recharts 展示经营趋势、分布和实验结果。
- 桌面端为深色侧栏办公布局，移动端使用抽屉导航。

### 5.2 后端

- FastAPI 负责 REST、WebSocket、认证和健康检查。
- Service 层封装生成、审核、预测、检索、聚类、指标和存储逻辑。
- SQLAlchemy Async 访问 PostgreSQL。
- Alembic 管理当前 8 个数据库迁移版本。
- Celery Worker 执行耗时任务，Celery Beat 执行定时任务。

### 5.3 数据与存储

主要数据实体：

- 商品、视觉方案和生成图片。
- 审核记录和质量问题。
- 预测记录和每日经营指标。
- A/B 实验。
- 商品向量。
- 品牌标准。
- 供应商评分和分析报告。
- 审计日志。
- 外部 Listing 映射。

MinIO 使用公开桶和私有桶区分展示资源与受保护资产；私有对象通过预签名 URL 访问。

## 6. 异步任务与实时进度

Celery 当前负责：

- 图片生成和质量处理。
- 审美/质量评估。
- 商品向量计算。
- 实验自动化和流量调整。
- 指标同步、样本回流和模型重训。

定时计划：

| 时间 | 任务 |
| --- | --- |
| 每天 02:00 | 同步每日指标 |
| 每周日 03:00 | 重训模型 |
| 每天 04:00 | 自动创建实验 |
| 每天 06:00 | 更新实验流量分配 |

生成进度通过 Redis Pub/Sub 和 WebSocket 推送；连接不可用时，前端继续使用轮询获取状态。

## 7. 安全设计

- OIDC Authorization Code + PKCE 和 JWT 会话。
- Redis 滑动窗口限流。
- 外部图片下载采用域名白名单、大小限制、重定向限制和私网地址阻断，降低 SSRF 风险。
- 指标写入使用独立机器 API Key。
- C2PA 用于 AI 内容来源签名和验证。
- Nginx 配置基础安全响应头、20 MB 上传限制和统一代理。
- 审计日志记录关键操作和请求链路。
- 生产环境启动时执行强制安全配置检查。

## 8. 可观测性

- 结构化应用日志。
- 请求数量、延迟和活跃请求 Prometheus 指标。
- 质量通过率、预测漂移、生成任务时长和队列长度等业务指标。
- Prometheus 保存时序数据。
- Grafana 展示监控看板。
- Flower 展示 Celery 任务和 Worker 状态。
- OpenTelemetry 可按环境配置导出分布式追踪。

## 9. 部署形态

仓库提供三类环境模板：

| 环境 | 用途 | 特点 |
| --- | --- | --- |
| Development | 本地开发和功能演示 | 可启用开发登录与生成 Mock，直接暴露调试端口 |
| Staging | 联调和压测 | 使用独立环境配置，保留调试端口和监控端口 |
| Production | 正式运行 | 强制认证、C2PA、指标 Key 和真实生成配置，业务流量经过 Nginx |

部署入口和具体命令参见 [README.md](./README.md)。

## 10. 当前功能边界

- 外部 AI 和电商平台能力只有在凭据、账号权限、网络和供应商服务均正常时才可用。
- 开发占位图只验证任务状态、存储、审核和前端交互，不代表真实模型输出。
- 系统不会在第三方通道不可用时伪造成功结果。
- 预测质量取决于已积累的真实指标和模型工件；冷启动阶段会使用明确的降级值。
- 公平性报告用于运营风险提示，不替代法律、合规或人口统计结论。
- 生产环境默认关闭 Swagger/ReDoc，并要求所有安全配置通过启动校验。
