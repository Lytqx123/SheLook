# SheLook 项目介绍

## 1. 为什么做这个项目

电商运营中有一个反复出现的困境：同一件商品，换一组主图，点击和转化可能差出 30% 以上。但大多数团队选图靠的是"感觉"——设计师凭经验、运营凭直觉、老板凭喜好。为什么不能像优化广告投放一样，用数据来指导视觉决策？

SheLook 的出发点就是这个问题。它不是另一个 AI 生图工具，而是尝试把商品视觉从"拍完就定"变成"可预测、可对比、可迭代"的决策过程。核心目标就三个：

- 让运营人员在商品上架之前，就知道不同视觉方案大概能跑出什么数据；
- 让审核过程从"看一眼凭感觉"变成有标准、有依据的结构化判断；
- 让线上的真实表现能够回流到模型中，推动下一次更好的决策。

## 2. 产品设计理念

### 2.1 不是替代人，是给人更好的决策依据

系统不会自动选择方案或自动发布商品。它做的事情是：给出预测分数、展示质量报告、跑实验对比、拉数据趋势。最终的决定权在运营、审核和增长团队手里，但他们的判断有了数据支撑。

### 2.2 把"视觉"变成可量化的维度

商品图的好坏通常被认为是主观的。但这个项目尝试把主观感受拆解为可测量的维度：合规性（有没有违规内容）、技术质量（清晰度、曝光、色彩是否达到基础标准）、审美表现（构图、主体突出度、商业可用性），以及最重要的——这张图在历史上的同类商品中表现如何。

### 2.3 闭环比单点能力重要

单纯能生成图片或单纯能做预测，都不足以改变工作流程。这个项目把生成、质量检查、效果预测、线上数据回流、模型更新串成一条完整的回路，让每一次决策都能为下一次决策积累信息。

## 3. 用户角色与工作流

| 角色 | 核心场景 | 使用的模块 |
| --- | --- | --- |
| 商品运营 | 上新时选择视觉方案、发起图片生成、查看效果预估 | 发品工作台、预测决策、商品管理 |
| 视觉审核 | 检查生成图片是否合规、质量是否达标、通过或驳回 | 审核工作台 |
| 供应商管理 | 评估供应商提供的素材质量、与品类平均水平对比 | 供应商分析 |
| 增长运营 | 对比不同视觉方案的线上表现、调整流量分配 | A/B 实验中心、数据看板 |
| 模型运营 | 导入线上数据、触发模型更新、管理模型版本 | 数据飞轮、指标数据管理 |

## 4. 功能模块

### 4.1 发品工作台

商品运营的主要入口。核心流程是：

1. **商品建档**：填写基础信息、类目、价格区间、目标市场。
2. **方案发现**：上传参考图或输入商品描述，系统会检索历史上表现不错的相似商品图作为视觉参考。这个环节的设计初衷是：与其让运营从零构思"什么样的图好"，不如先看看同类商品里跑得好的图长什么样。
3. **方案推荐**：结合商品属性、视觉特征和同类商品的历史数据，给出综合推荐排序。推荐结果会标注参考依据（例如"该类目 Top 20% 商品倾向使用白底棚拍"），帮助运营理解决策逻辑。
4. **图片生成**：确认方案后发起异步生成任务。这个环节做了两个设计取舍：一是生成任务统一异步处理，避免运营干等；二是进度通过 WebSocket 实时推送，生成完一张就能看到一张，不用等全部完成。
5. **生成后质检**：生成完成后自动跑图文匹配检查（生成的图和商品描述是否对得上）、审美评分、合规筛查。运营可以在同一个面板上看到每张图的质检结果。

### 4.2 预测决策

这个模块解决的核心问题是"在投放之前，有没有办法预估效果"。系统把商品属性、图片视觉特征和同类商品的历史表现组合在一起，输出几个关键指标：

- **预估 CTR**：与同类目平均水平对比的相对值，以及一个置信区间。
- **爆款概率**：基于历史数据，判断这张图的 CTR 有多大可能进入 Top 20%。
- **退货风险**：重点关注图片是否可能造成"实物与图片不符"的预期落差。

对同一个商品可以批量对比多个方案的预测结果，运营可以直接看到"方案 A 比方案 B 预估 CTR 高 15%，但方案 B 的退货风险更低"这样的比较信息。

模型冷启动阶段或样本不足时，系统会返回降级值并明确标注"预测仅供参考"，不会给出看起来很精确但实际上没有依据的数字。

### 4.3 审核工作台

审核分为三个层级，由浅入深：

- **L1 合规**：自动检查内容安全、品牌规范、元数据完整性。这一层的设计原则是"宁可误杀也不漏放"——对不确定的项标记为待人工复核，不自动通过。
- **L2 技术质量**：检查清晰度、曝光、对比度、色彩均衡、白边等像素级指标。这些都有明确的量化阈值，机器判断的准确率很高。
- **L3 审美评估**：这是最主观的一层，系统不会给"美不美"下结论，而是从构图、主体突出度、背景整洁度、商业可用性等维度给出参考评分。审核人员结合这些维度做最终判断。

审核队列每 10 秒自动刷新，支持批量自动审核和逐张人工决策。审核决策会触发关联数据的更新——审核通过的图片可以在电商平台上架，审核驳回的图片可以追溯驳回原因。

### 4.4 公平性分析

确保不同肤色、不同市场的商品图分布是合理的。系统会统计当前图片库中各分类的分布比例，与运营配置的基线对比，标记出明显偏离的情况。对单个方案也可以跑偏差检查。

公平性基线是运营配置项，不是法律结论。正式使用前需要由合规和法务团队确认。

### 4.5 A/B 实验中心

视觉方案的线上验证工具。支持创建对比实验（方案 A vs 方案 B），按日期等维度拆分查看真实 CTR、CVR 等指标，并通过统计检验判断差异是否显著。实验支持手动创建和自动创建两种模式。

流量分配采用 UCB 策略：表现好的方案会逐步获得更多曝光，但始终保留最低探索流量，避免过早收敛到次优解。

### 4.6 数据飞轮

这个模块是整个系统的"学习引擎"。每天凌晨会自动聚合前一天各平台上的经营数据（CTR、CVR、订单等），按配置的阈值自动标注正样本和负样本。当样本积累到一定量级后，可以手动触发模型重训。

模型版本管理支持查看当前版本、历史版本，以及回滚到任意历史版本。这样即使模型更新不如预期，也能快速退回。

### 4.7 数据看板

经营侧和数据侧的信息汇总。包含：
- 整体指标概览：生成量、审核通过率、CTR、CVR、退货率。
- CTR 近 30 天趋势。
- 各市场的表现对比（不同市场的消费者对视觉风格的偏好可能差别很大）。
- 风格标签的分布和使用情况。
- 模型自身的健康度指标：AUC、高预测命中率、人工复核比例。

### 4.8 聚类分析

从视觉维度理解图片库的工具。使用聚类算法将所有商品图的视觉特征分组，每个组（簇）汇总样本量、平均 CTR、退货率、主要类目和标签。这个模块的典型场景是：运营发现某个簇的退货率异常高，点进去一看，原来是"深色背景 + 暖光"的拍摄风格在深色服装上容易产生色差。

### 4.9 供应商分析

面向供应商质量管理的模块。输入供应商提供的图片 URL，系统会返回一份分析报告，包含：质量评分、审美评分、品牌一致性、预测效果和退货风险预估，以及与品类平均水平的对比。供应商的历史报告可以按 ID 查询，便于追踪质量变化趋势。

### 4.10 指标数据

经营指标的管理入口。支持通过 JSON 批量写入每日指标数据（按图片、日期和平台维度幂等写入），也支持同步 Shopee 和 Amazon 平台指标。数据写入后有统计面板可查看数据总量、覆盖范围和日期跨度。

### 4.11 平台导出与视频生成

生成的图片需要适配不同平台的规范。当前支持 Amazon、天猫、TikTok Shop、TikTok 方图和 Shopify 五种规格的自动裁剪和 AI 标识处理。视频生成支持 Kling 和 Runway 两个通道，优先使用已配置的可用通道。

## 5. 技术路径

系统底层的技术栈围绕一个核心思路搭建：用视觉特征作为桥梁，串联起"理解图片 → 生成图片 → 评估图片 → 预测效果 → 学习反馈"这条链路。

视觉理解层面，使用多模态模型将图片映射为向量表示，同一套向量能力同时支撑以图搜图、图文匹配、聚类分析和标签提取等场景。方案推荐环节融合了视觉相似度、商品属性匹配和历史数据参考，给出综合排序。

效果预测是产品里技术挑战最大的部分。需要把商品的结构化属性、图片的视觉特征和质量评分、同类商品的历史经营数据组合在一起，输出 CTR 预估、爆款概率和退货风险。模型不可用或样本不足时，使用明确的降级策略，不在预测结果中伪造置信度。

## 6. 安全与运维

### 安全
- 支持 OIDC 认证（Authorization Code + PKCE）和开发环境的简化登录流程。
- Redis 滑动窗口限流，生产环境默认 600 次/60 秒。
- 外部图片下载有域名白名单、大小限制和 SSRF 防护。
- 指标写入接口使用独立 API Key，与用户认证分离。
- AI 生成内容支持 C2PA 来源签名，生产环境强制开启。
- 所有关键操作记录审计日志，支持按请求 ID 追踪完整链路。

### 可观测性
- Prometheus 采集请求量、延迟、业务指标（质量通过率、预测漂移、队列长度等）。
- Grafana 看板展示系统健康和业务状态。
- Flower 查看异步任务执行情况。
- 可选 OpenTelemetry 分布式追踪。

## 7. 部署方式

项目通过 Docker Compose 部署，包含 PostgreSQL、Redis、MinIO、后端 API、Celery Worker/Beat、Flower、前端、Nginx、Prometheus 和 Grafana 共 11 个核心服务。提供开发、预发和生产三套环境模板，支持 Windows 和 Linux/macOS 一键部署。

## 8. 当前边界

- 图片生成和视频生成依赖第三方 API，可用性取决于外部服务的配额和状态。
- 预测模型的效果与已积累的真实指标数据正相关；冷启动阶段使用降级策略。
- 公平性分析是运营工具，不替代合规和法律审查。
- 生产环境的安全和认证配置必须通过启动校验。
