# SheLook 项目介绍

## 1. 为什么做

电商运营里有个反复出现的问题：同一件商品换组主图，点击和转化能差 30% 以上。但大部分团队选图还是靠"感觉"——设计师凭经验、运营凭直觉、老板凭喜好。

所以就试着做个工具，把商品视觉从"拍完就定"变成"可预测、可对比、可迭代"的过程。三个核心目标：

- 上架前就知道不同视觉方案大概能跑出什么数据；
- 审核从"看一眼凭感觉"变成有标准的结构化判断；
- 线上真实表现能回流到模型，推动下次更好的决策。

## 2. 产品设计

### 2.1 不是替代人，是给人更好的依据

系统不会自动选方案或自动发布商品，只是给出预测分数、质量报告、实验对比和数据趋势。最终拍板的是运营和审核团队，但判断有了数据支撑。

### 2.2 把"视觉"拆成可量化的维度

商品图好不好看通常是主观的。这个项目试着拆成可测量的维度：合规性、技术质量（清晰度/曝光/色彩）、审美表现（构图/主体/商业可用性），以及最重要的——这张图在同类商品历史数据里表现怎么样。

### 2.3 闭环比单点能力重要

光能生图或光能预测不够——把生成、质检、预测、数据回流、模型更新串成闭环，每次决策都能为下次积累信息。

## 3. 用户角色与工作流

| 角色 | 核心场景 | 用的模块 |
| --- | --- | --- |
| 商品运营 | 上新时选方案、发起生图、看效果预估 | 发品工作台、预测决策、商品管理 |
| 视觉审核 | 检查生成图合规/质量，通过或驳回 | 审核工作台 |
| 供应商管理 | 评估供应商素材质量 | 供应商分析 |
| 增长运营 | 对比不同方案线上表现、调流量 | A/B 实验、数据看板 |
| 模型运营 | 导入数据、触发重训、管理版本 | 数据飞轮、指标管理 |

## 4. 功能模块

### 4.1 发品工作台

运营的主要入口，核心流程：

1. **商品建档**：填基础信息、类目、价格、目标市场。
2. **方案发现**：上传参考图或输描述，检索历史表现好的相似商品图当参考——与其从零构思"什么图好"，不如先看看同类跑得好的图长啥样。
3. **方案推荐**：结合商品属性、视觉特征和历史数据，给出综合排序并标注参考依据（比如"该类目 Top 20% 倾向白底棚拍"）。
4. **图片生成**：异步提交，WebSocket 实时推进度，生成完一张看一张不用等全部。
5. **生成后质检**：自动跑图文匹配、审美评分、合规筛查，同面板看结果。

### 4.2 预测决策

核心问题："投放前能不能预估效果"。把商品属性、图片特征和历史表现拼一起，输出：

- **预估 CTR**：和同类目平均对比的相对值 + 置信区间
- **爆款概率**：CTR 进 Top 20% 的可能性
- **退货风险**：图片会不会造成"实物与图片不符"

同一商品可批量对比多个方案。冷启动或样本不足时返回降级值并标注"仅供参考"，不伪造精确数字。

### 4.3 审核工作台

三级审核：

- **L1 合规**：内容安全、品牌规范、元数据。原则是"宁可误杀不放过"——不确定的标待人工复核。
- **L2 技术质量**：清晰度、曝光、对比度、色彩等像素指标，有明确阈值，机器判断准确率很高。
- **L3 审美评估**：最主观的一层，从构图、主体、背景、商业可用性等给出参考评分，审核人员最终判断。

审核队列每 10 秒自动刷新。驳回可追溯原因。

### 4.4 公平性分析

统计不同肤色、市场的分布比例，和运营配置的基线对比，标记明显偏离。支持单方案偏差检查。公平性基线是运营配置项，不是法律结论。

### 4.5 A/B 实验

视觉方案线上验证。支持手动/自动创建，按日期等维度看 CTR/CVR，跑统计检验判断差异是否显著。

流量分配用 UCB：表现好的逐步获更多曝光，但保留最低探索流量避免过早收敛。

### 4.6 数据飞轮

整个系统的"学习引擎"。每天凌晨自动聚合经营数据（CTR/CVR/订单等），按阈值标注正负样本。积累到量后手动触发重训。

模型版本管理支持当前/历史/回滚。

> 先记一下：重训流程目前还是手动触发，后续考虑要不要做成自动的

### 4.7 数据看板

经营和数据汇总：
- 整体指标：生成量、审核通过率、CTR、CVR、退货率
- CTR 近 30 天趋势
- 各市场表现对比
- 风格标签分布
- 模型健康度：AUC、高预测命中率、人工复核比例

### 4.8 聚类分析

用聚类算法把商品图按视觉特征分组，每个簇汇总样本量、平均 CTR、退货率等。典型场景：发现某簇退货率异常高，点进去发现"深色背景+暖光"在深色服装上容易产生色差。

### 4.9 供应商分析

输入供应商图片 URL，返回质量评分、审美评分、品牌一致性、预测效果和退货风险预估，以及和品类平均的对比。历史报告按 ID 可查。

### 4.10 指标数据

经营指标管理。支持 JSON 批量幂等写入（按图片/日期/平台），支持同步 Shopee 和 Amazon。有统计面板看数据总量和覆盖范围。

### 4.11 平台导出与视频生成

适配 Amazon、天猫、TikTok Shop、Shopify 五种规格的自动裁剪。视频生成支持 Kling 和 Runway，优先用可用通道。

## 5. 技术路径

底层思路：用视觉特征当桥梁，串起"理解→生成→评估→预测→反馈"整条链路。

视觉理解用多模态模型把图片映射为向量，同一套向量能力同时支撑以图搜图、图文匹配、聚类分析和标签提取。方案推荐融合视觉相似度、属性匹配和历史参考。

效果预测是技术挑战最大的部分——需要把结构化属性、视觉特征、质量评分和历史数据拼一起，输出 CTR/爆款概率/退货风险。模型不可用时降级，不伪造置信度。

## 6. 安全与运维

### 安全
- 支持 OIDC（Authorization Code + PKCE）和开发环境简化登录
- Redis 滑动窗口限流，默认 600 次/60 秒
- 外部图片下载有白名单、大小限制和 SSRF 防护
- 指标写入用独立 API Key
- 生图支持 C2PA 来源签名，生产强制开启
- 关键操作记录审计日志，按请求 ID 追踪完整链路

### 可观测性
- Prometheus 采集请求量、延迟、质量通过率、预测漂移、队列长度等
- Grafana 看板（系统健康/模型监控/业务质量三套）
- Flower 看异步任务
- 可选 OpenTelemetry 追踪

## 7. 部署

Docker Compose 部署，共 11 个核心服务：PostgreSQL、Redis、MinIO、后端、Celery Worker/Beat、Flower、前端、Nginx、Prometheus、Grafana。提供 dev/staging/prod 三套环境模板，Windows 和 Linux/macOS 一键部署。

## 8. 当前边界

- 生图和视频依赖第三方 API
- 预测效果和线上数据量正相关，冷启动降级
- 公平性分析是运营工具，不替代合规和法律审查
- 生产环境安全和认证必须通过启动校验
