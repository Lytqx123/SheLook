# 第五阶段：企业体验、运行可观测性与安全治理（当前状态）

> 文档基线：2026-07-23。页面可见性不替代服务端鉴权；密钥只写、审计和租户范围由后端执行。

本文说明当前版本已经落地的企业使用体验与生产运行治理，并明确哪些能力仍需由具体部署环境完成。

2026-07-22 复核中还修正了 Grafana 的子路径代理：Grafana 使用完整公开 `GRAFANA_ROOT_URL`，Nginx 在转发时剥离 `/grafana/` 前缀；预发和生产环境会拒绝缺少以 `/grafana/` 结尾 HTTPS URL 的配置。观测页面可访问不等于告警与值班流程已经完成。

## 产品体验已经以“视觉运营活动”为中心

SheLook 不再把用户引导为在商品、方案、生成、审核、预测和实验模块之间来回切换。日常运营入口以“视觉运营活动”为主线，活动将目标市场、商品、素材方案、生产进度、审核、预测、实验、下一步建议和复盘洞察放在同一业务上下文中。首页则优先呈现待决策事项、审核阻塞、失败任务、运行中的实验和经营信号；趋势图与模型信息退到解释层，而不是替代行动。

前端按日常运营、经营决策、分析与治理组织导航。活动详情呈现阶段、经营目标、关联素材、风险/置信度、实验状态和可复用洞察；预测页面把分数转换为包含依据、风险和下一动作的决策卡片；复盘页面将已验证或待验证的结果写回活动洞察。数据表统一使用分页、筛选、状态标签和克制的办公化布局，不以装饰性卡片替代真实数据密度。

## 角色化体验与服务端边界

运营人员、审核人员、分析人员、供应商、管理者和只读成员拥有不同的首页文案、优先事项、导航可见性和行动入口：运营关注增长与活动推进，审核关注人工队列，分析关注实验和异常信号，供应商关注交付质量，管理者关注跨模块经营与治理。

前端的角色判断只用于减少无关入口、降低学习成本；真正的读写授权仍由后端权限中间件、路由校验和租户数据边界执行。无权限用户不能通过手工修改 URL、隐藏按钮或重放接口请求绕过服务端校验。

## 当前可观测性

- API 输出请求计数、路由级延迟、活跃请求和业务指标，并提供 `/metrics`；可通过 `METRICS_API_KEY` 对指标端点加保护。
- 请求会携带 `X-Request-ID` / `X-Audit-Trace-ID` 和响应耗时，结构化日志可关联用户操作、任务、审计与异常。
- Prometheus、Grafana 和 Flower 已包含在 Compose 运行基线中；Grafana 预置系统健康、质量和模型监控看板。
- OpenTelemetry 依赖和开关已提供，只有在配置 OTLP 端点并启用后才会真正导出链路追踪。
- 后端提供存活、就绪和综合健康检查；Kubernetes API 使用就绪/存活探针、滚动更新、PDB 和 HPA。

这些能力提供了采集和展示基础。仓库当前没有 Alertmanager、值班轮转、工单升级或云监控集成，因此告警阈值、通知渠道和事故响应仍需由实际运行团队配置并演练。

## 当前安全与内容治理

- 企业 OIDC、飞书 OAuth、成员资格校验、会话撤销、租户 RLS、配额和限流共同保护身份与数据边界。
- API 提供 CORS 配置、安全响应头、受控远程图片下载规则、请求大小/超时限制和审计上下文；外部图片读取不会接受任意本地路径。
- 素材先进入私有对象桶；对外发布时才复制到公开桶。平台格式导出会按平台规格处理，并可添加 AI 生成标识。
- C2PA 内容凭证在开发/非生产环境以配置开关提供；生产启动会强制要求启用并要求有效的证书和私钥。只有签名链路实际可用时，才能对外宣称素材带有内容凭证。
- 公平性、模型监控、供应商深度分析和审计属于分析/治理层，默认不挤占运营主路径；治理数据应作为风险提示和复核依据，而不是自动替代业务负责人决策。

## 部署与发布基线

本地和预发可使用 Compose，生产覆盖配置会收紧端口暴露、移除源码挂载并设置资源限制。Kubernetes 目录提供 API、队列 Worker、迁移 Job、运行配置、PVC、PDB 和 HPA 的基础清单；镜像、Secret、数据库角色、入口、证书和云资源需要在受保护的部署环境中补齐。

Kubernetes 的运行配置已明确关闭生成 Mock。使用 Compose 的生产覆盖时，`ALLOW_GENERATION_MOCKS` 继承基础文件的默认值，**必须**在生产环境文件中显式设为 `false`；否则不能把该 Compose 部署视为真实 AI 结果环境。

## 当前边界

- 前端已实现角色化工作台和活动主线，但没有独立的 BFF 层，也没有为每一种角色建立完全独立的数据模型；优先复用同一 API 与权限边界。
- 权限隐藏、表格布局和看板排序不等于业务 SLA。大规模并发、可访问性审计和跨浏览器兼容性仍需要在目标组织的真实设备与数据量下验收。
- 预置 Grafana 看板不等于已完成告警闭环；没有告警接收方、处置手册和演练记录时，不应称为完整 SRE 体系。
- 供应商、平台与 AI 提供商的凭据均是环境级机密，不能写入仓库、ConfigMap、前端包或普通日志。
