# 第一阶段：基线、架构与工程治理（当前状态）

> 文档基线：2026-07-23。以迁移头 `018` 和已部署 Compose 开发栈为当前工程基线。

本文不是历史实施计划，而是 SheLook 当前版本的工程基线。后续的功能、容量和部署变更，都应以这里的边界为准。

2026-07-23 已完成本机 development 运行复核：Compose 服务栈、迁移 `018`、对象存储初始化、API/Nginx 就绪检查、前端生产构建和后端静态检查均通过。该结果证明可运行性，不构成目标环境的容量或可用性承诺。

## 当前结论

SheLook 采用“模块化单体 + 独立异步 Worker”的架构：前端负责运营工作台，FastAPI 提供统一业务 API，PostgreSQL/pgvector 保存交易数据与向量数据，Redis 承担缓存、消息代理和任务结果，MinIO 保存素材对象，Celery 按生成、编排、模型与分析队列执行后台任务。它不是微服务集群；领域在代码和数据模型中分层，仍以一个后端应用为发布与治理中心。

当前面向业务的主线是视觉运营活动：以活动聚合商品、目标市场、视觉方案、素材、审核、预测、实验和复盘洞察，形成“目标定义—生产—质量门禁—预测—实验—学习”的可追踪链路。商品、图片、实验和指标表仍是各自领域的事实来源；活动只保存关联关系和决策上下文，不复制或取代这些事实数据。

## 已固定的领域边界

| 领域 | 当前职责 | 主要承载方式 |
| --- | --- | --- |
| 视觉运营活动 | 串联本次经营目标、阶段、行动建议、时间线与洞察 | `visual_operation_campaigns`、`campaign_insights` 与 `/api/v1/campaigns` |
| 内容生产 | 商品、视觉方案、图片/视频生成、素材存储与导出 | API + 生成 Worker + MinIO |
| 质量与风险 | 自动评估、人工审核、合规与问题追踪 | 审核记录、审核队列与审计日志 |
| 经营决策 | CTR/爆款/退货风险预测、A/B 实验、归因与指标回流 | 预测、实验、指标与飞轮任务 |
| 企业治理 | 租户、成员、角色、配额、功能开关、审计与任务恢复 | 组织模型、权限中间件、RLS、工作流表 |

跨领域调用通过 API、服务层或异步任务完成。新功能不应直接跨模块修改对方的业务表；需要联动时，应复用已有聚合、服务接口或带审计上下文的任务事件。

## 数据、版本与变更护栏

- 数据库迁移由 Alembic 管理，当前迁移头为 `018`；`015_visual_operation_campaigns` 增加视觉运营活动及活动洞察，`016` 至 `018` 依次补齐企业连接、版本化运行时配置及经营事实/真实效果/预测反馈数据模型，并在 PostgreSQL 上启用相应的租户行级安全策略。
- 生产部署应把迁移权限与运行权限分开：迁移作业使用 `DATABASE_MIGRATION_URL` 的受控 schema-owner 账号，API 与 Worker 使用非超级用户、非 `BYPASSRLS` 的运行账号。
- 对已租户化的业务表，应用层会自动附加当前租户过滤条件，并在 PostgreSQL 事务中写入 `app.tenant_id` 供 RLS 二次校验。新增业务表必须明确是否继承租户范围。
- 接口优先新增字段和兼容路由。活动接口同时提供 `/api/v1/campaigns` 与兼容别名 `/api/campaigns`，避免前端联调期间因路由切换中断；实现上将路由、共享规则和详情读模型拆分，以降低单一 API 模块继续膨胀的维护风险。
- 每个请求带有可回传的请求/审计追踪 ID；生成、审核、指标和任务的关键写操作保留审计上下文。

## 可重复的工程基线

仓库提供开发、预发和生产 Compose 覆盖配置，以及 Kubernetes 基础清单。GitHub Actions 已配置以下质量门禁：后端 Ruff 与测试、PostgreSQL 迁移及隔离校验、前端 Lint、TypeScript 类型检查和生产构建。容器提供健康检查；后端提供存活与就绪探针，Kubernetes API 清单配置滚动更新、Pod 中断预算和 HPA。

本地验证的最小闭环是：迁移到当前版本、启动 API/Worker/前端、执行后端测试与前端构建，再以真实权限访问活动、审核、预测、实验与任务中心。性能或安全结论必须附带环境、数据规模、并发模型和配置，不能只记录单一吞吐量数字。

## 当前治理边界

- 代码和部署骨架已经具备，但仓库没有一份可直接外推到任意硬件或云环境的固定 SLO/容量结论；性能阈值需由预发压测报告确认。
- Kubernetes 清单是后端与 Worker 的运行基线，不等同于已经在某个生产集群完成上线验证；镜像仓库、Secret、入口、备份和告警路由需由实际环境补齐。
- 领域已经分层，但尚未拆分为独立微服务。没有明确的容量、团队边界或故障隔离收益时，不应为了“服务化”而拆分。
- 开发环境允许关闭认证或使用受控模拟配置；生产环境必须启用认证、真实密钥、受限数据库角色和 `ALLOW_GENERATION_MOCKS=false`。
