## 当前事实基线

- 当前后端迁移头为 `019`：运行时配置、统一经营事实、真实效果事实、预测快照、反馈标签和租户级外部 API 配置已存在。
- 店小秘连接、配置校验和同步任务框架已部署；真实网络同步必须等待企业获得的供应商接口契约，当前不会假定端点或签名规则。
- 真实 CTR 的唯一可用口径是同一实体和时间粒度上的 `sum(clicks) / sum(impressions)`；ERP 订单、库存和履约数据不是 CTR 替代品。
- 业务运行时配置和业务供应商 API 可经 Web 维护；Kling、Runway、Replicate、Gemini、Shopee 与 Amazon 的凭据按租户加密保存、只写且不回显。数据库、消息队列、JWT、加密根密钥、TLS、监控入口等启动级安全参数仍由部署 Secret 管理。

## 文档维护规则

1. 新增迁移、路由、页面或 Worker 时，同时更新 `后端.md`、`前端.md`、对应重构方案和 README 入口。
2. 教学文档中的示例参数、流程图和供应商名称必须标明是说明性内容；不能被视为生产配置或接口承诺。
3. 所有部署、性能和可用性结论都必须带环境和验证范围，不得由本地 Compose 健康状态外推为生产 SLA。
4. 涉及第三方授权、数据口径、租户覆盖优先级或安全根密钥的冲突，必须记录为待决项，不得由代码隐式选择。

## 本轮审计交付标准

- 枚举仓库内全部受版本控制或交付范围内的 Markdown 文件，并在每份文件中写入当前基线说明。
- 对源码、配置、部署、测试和依赖进行分层检查；缺陷以可复现证据、修复和验证结果闭环。
- 对无法安全推断的外部协议保留明确门禁和下一步输入清单，而不是实现猜测性调用。
