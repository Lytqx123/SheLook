# 第二阶段：多租户、组织与权限底座（当前状态）

> 文档基线：2026-07-23。权限检查、应用层租户范围和 PostgreSQL RLS 共同构成当前隔离边界。

本文说明 SheLook 当前已经落地的数据边界和授权机制。它描述的是系统行为，不把尚未覆盖的企业治理能力写成已完成能力。

> 复核说明（2026-07-22）：本机部署确认了数据库、缓存和对象存储依赖连通；真实企业 OIDC/飞书登录、跨租户拒绝和成员撤销仍必须在受控预发环境以真实身份映射验收。

## 当前结论

SheLook 的数据隔离单位是租户。商品、方案、素材、审核、预测、实验、指标、供应商分析、审计、工作流和视觉运营活动均带有 `tenant_id`；应用会在请求和后台任务中建立租户上下文，ORM 自动限制租户化模型的查询，新写入记录自动继承当前租户。在 PostgreSQL 上，RLS 作为数据库层的第二道边界，生产启动会拒绝超级用户或拥有 `BYPASSRLS` 的运行账号。

组织管理由租户、组织单元、成员和配额组成。组织单元可记录父子关系与类型；成员保存角色、附加权限和可见组织单元 ID；配额覆盖 API 请求频率、图片生成并发、月度生成次数、存储容量和 AI 预算。当前实现以租户为强制数据隔离边界，组织单元范围用于成员上下文和后续扩展，尚未成为所有业务查询的全量 ABAC 过滤条件。

## 授权模型

| 层级 | 当前机制 | 作用 |
| --- | --- | --- |
| 身份 | 开发 JWT、通用企业 OIDC、飞书 OAuth 授权码登录 | 识别用户并建立本地会话 |
| 成员资格 | `tenant_memberships` 中的有效成员记录 | 外部身份不能仅凭企业信息进入租户，必须是已受邀成员 |
| 角色 | `admin`、`operator`、`reviewer`、`analyst`、`supplier`、`viewer` | 提供默认权限集合 |
| 附加权限 | 成员的 `permissions` 列表 | 在角色默认值上增补细粒度操作 |
| API 执行点 | 授权中间件与路由级校验 | 在服务端拒绝无权读写，前端隐藏不是安全边界 |
| 数据边界 | ORM 租户范围 + PostgreSQL RLS | 防止应用查询遗漏时的跨租户读取与写入 |

常用权限已覆盖商品、生成、审核、分析、实验、供应商、审计、模型和任务治理。视觉运营活动接口允许拥有活动、商品、生成、实验、分析或审核相应权限的成员按职责读取或推进活动；具体动作仍由后端校验，而不是由页面入口决定。

## 企业登录与会话安全

- 通用 OIDC 使用 Discovery、授权码 + PKCE、`state`、`nonce` 和 ID Token 校验；多租户场景必须用显式的 `OIDC_TENANT_CLAIM_MAP` 将外部组织映射到本地租户。
- 飞书走 OAuth 2.0 授权码流程，不把飞书当作 OIDC Provider。飞书 `tenant_key` 必须命中受控映射或单租户允许列表，浏览器回调参数不能直接指定本地租户。
- 登录成功后仍会校验本地租户和有效成员资格。成员禁用、角色或权限变更会写入 Redis 会话撤销标记；生产环境 Redis 不可用时相关校验与成员变更按失败关闭处理，避免旧权限继续生效。
- 开发模式可关闭认证并使用默认开发身份，便于本地联调；这不是生产授权方案。生产启动要求至少配置一个完整的企业登录提供方，并使用 HTTPS 回调地址。

## 配额、限流与发布控制

租户 API 限流由 Redis 执行，并读取租户的每分钟请求配额。图片生成提交前会在租户范围内串行检查活跃任务数、月度生成上限与月度 AI 预算；用量以预留/完成/失败等状态记录，避免同一请求重复占用预算。`ai_generation`、`video_generation`、`automated_experiments` 是当前可按租户关闭的功能开关，适合异常止损和分批启用。

## 运行与审计要求

- API、Worker、缓存键、对象存储路径和异步事件都必须使用当前租户上下文；后台任务不能依赖隐式“全局默认租户”。
- PostgreSQL 的运行账号必须不是表所有者，且不具备超级用户或 `BYPASSRLS` 权限；schema-owner 账号只能交给迁移作业。
- 组织成员、权限、配额和功能开关变更应通过组织管理 API 或受控运维流程执行，并保留审计记录。
- 对导出、第三方连接和新建外部身份映射，需要在实际组织场景中补充权限回归测试，尤其是跨租户和成员停用后的即时失效测试。

## 当前边界

- 组织单元模型与 `unit_ids` 已存在，但品牌、店铺、市场、类目等属性级范围还没有被统一强制到每一条业务查询；当前的可靠隔离单位仍是租户。
- 没有 SCIM 自动开通/离职同步，也没有 SAML 协议实现；企业可使用已实现的 OIDC 或飞书登录，并由管理员维护本地成员关系。
- 配额覆盖 API、生成和预算等核心资源，但不等于完整的计费、合同或成本分摊系统。
- SQLite 用于本地和测试兼容场景，不提供 PostgreSQL RLS；涉及真实多租户安全验证时必须在 PostgreSQL 上执行。
